# BIP-0100: Dynamic maximum block size by miner vote

**Status:** speculative
**Last updated:** 2026-06-07
**Sources:** [[bip-0100]]

Layer: Consensus (hard fork) · Typ: Specification · Status: Closed · Datum: 2015-06-11
**Autoren:** Jeff Garzik, Tom Harding, Dagur Valberg Johannsson

## Summary

Replace the static 1M block size hard limit with a hard limit set by coinbase vote, conducted on the same schedule as difficulty retargeting.
