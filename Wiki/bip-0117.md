# BIP-0117: Tail Call Execution Semantics

## Summary

In conjunction with BIP-116 (MERKLEBRANCHVERIFY), this BIP allows a script to commit to a practically unbounded number of code pathways, revealing only the actual code pathway used at spend time.

### Zweck

BIP-0117 introduces "tail call" execution semantics for Bitcoin Script: when a script reaches its end with a single serialised script on the stack, execution continues by evaluating that serialised script as a new script. This allows a top-level script to act as a dispatcher — it verifies a Merkle proof (via BIP-116) and then "tail calls" into the actual spending script.

Together with BIP-116, this enables a clean MAST implementation: the output commits to a Merkle root; at spend time, the spender provides a branch (the actual script) and a proof; BIP-116 verifies the proof; BIP-117 executes the branch. The unexecuted branches are never revealed.

Like BIP-116, BIP-117 was made redundant by Taproot's integrated MAST. Taproot achieves the same goals (hidden script branches, efficient spend) at the protocol level with better privacy properties.
