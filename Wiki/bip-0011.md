# BIP-0011: M-of-N Standard Transactions

## Summary

This BIP proposes M-of-N-signatures required transactions as a new 'standard' transaction type.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Gavin Andresen  
**Veröffentlicht:** 2011-10-18  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0011.mediawiki

### Zweck

BIP-0011 made M-of-N multisig a standard transaction type in Bitcoin. Before this, `OP_CHECKMULTISIG` outputs existed in the scripting language but weren't in the relay rules, so most nodes would drop them as non-standard and miners wouldn't include them. BIP-11 added bare multisig (up to 3-of-3) to the standard relay policy.

The tradeoff with bare multisig: the full script (with all public keys) sits in the output scriptPubKey, which bloats the UTXO set and makes the sender responsible for knowing the recipients' keys. P2SH (BIP-16) later solved this by moving the script to the spending transaction. Most modern multisig uses P2SH or P2WSH wrappers rather than bare BIP-11 multisig, but BIP-11 established the foundational opcode usage.
