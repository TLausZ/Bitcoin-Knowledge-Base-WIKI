# BIP-0128: Timelock-Recovery Storage Format

## Summary

This document proposes a standard format for saving timelock-recovery plans, allowing different wallets to generate them and different services to monitor and execute them.

### Zweck

BIP-0128 proposes a standardised file format for "dead man's switch" or inheritance recovery plans built on Bitcoin timelocks (BIP-65 CLTV and BIP-68 CSV). The scenario: a user sets up a UTXO spendable by a recovery key only after a timelock expires. If the user regularly "refreshes" (spends and recreates the UTXO with a new timelock), the recovery key never activates. If the user dies or becomes incapacitated, the timelock eventually expires and the recovery party can claim the funds.

The BIP defines a portable file format storing the recovery plan: which UTXOs are covered, the timelock parameters, the recovery keys, and instructions for monitoring. Without standardisation, each implementation invents its own format, making it impossible to use one wallet to set up a plan and a different service to monitor and execute it.

A 2026 Draft, this reflects growing interest in Bitcoin inheritance solutions.
