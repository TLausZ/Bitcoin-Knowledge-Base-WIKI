# BIP-0348: CHECKSIGFROMSTACK

## Summary

This BIP proposes CHECKSIGFROMSTACK (CSFS), an opcode that verifies a Schnorr signature against an arbitrary message on the stack (rather than the spending transaction).

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)

### Zweck

Bitcoin's existing signature opcodes (OP_CHECKSIG, OP_CHECKSCHNORR) verify signatures over the spending transaction hash. CHECKSIGFROMSTACK instead verifies a signature over an *arbitrary message* that is already on the stack at execution time.

Combined with OP_TXHASH (BIP-346), CSFS enables transaction introspection: push the current transaction hash onto the stack (via OP_TXHASH), require a signature over it from a specific key, and verify that signature with CSFS. The key can be an oracle, a time-locked delegation, or a covenant enforcer.

Applications: oracle-signed condition covenants (a script that only executes if an oracle signs off on current price), delegation (a key that can sign spending authorisations without having the private key for the UTXO), and composable covenant logic that doesn't require pre-committed transaction templates.
