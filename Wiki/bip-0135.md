# BIP-0135: Generalized version bits voting

## Summary

This BIP proposes generalized version bits voting to provide more flexible soft fork activation than BIP-9's fixed parameters.

## Body

**Typ:** Informational  
**Layer:** unspecified  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Themen:** bips, mining
**Autor(en):** Sancho Panza  
**Veröffentlicht:** 2017-03-29  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0135.mediawiki

### Zweck

BIP-0135 proposed making BIP-9's version bits activation parameters configurable per-deployment. BIP-9 hardcoded the activation threshold at 95% and the signalling window at 2016 blocks; BIP-135 would allow each soft fork deployment to specify its own threshold, window size, and minimum activation delay.

The motivation was that different changes might justify different thresholds. A very conservative change with no real controversy might not need 95%; a highly contested change might want a higher bar or a longer discussion period. The proposal was closed as the community moved toward BIP-8 as the preferred alternative to BIP-9, which addressed the core problem (miner veto power) differently.
