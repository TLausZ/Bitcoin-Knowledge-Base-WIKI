# BIP-0019: M-of-N Standard Transactions (Low SigOp)

## Summary

This BIP proposes M-of-N-signatures required transactions as a new 'standard' transaction type using the existing scripting system without significant modifications.

### Zweck

BIP-0019 was a competing multisig proposal to BIP-11, motivated by a specific technical concern: signature operation (sigop) counting. `OP_CHECKMULTISIG` as used in BIP-11 counted each key in the script as a sigop, not each actual signature checked. A 3-of-5 multisig would count as 5 sigops even if only 3 signatures were verified. This bloated the sigop count for blocks, which is subject to a consensus limit.

BIP-19 proposed an alternative script structure that counted sigops based on the number of signatures actually required (M), not the total keys (N). The approach was closed in favour of P2SH (BIP-16), which sidesteps the sigop problem by moving the script out of the output entirely — the full script is only revealed at spend time, and sigop counting uses the script's actual contents.
