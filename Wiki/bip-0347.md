# BIP-0347: OP_CAT in Tapscript

## Summary

This BIP re-enables OP_CAT as a Tapscript opcode, allowing concatenation of two stack values.

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)  
**Status:** Complete (abgeschlossen)

### Zweck

OP_CAT was originally in Bitcoin but was disabled by Satoshi in 2010, reportedly due to concerns about stack memory abuse (concatenating values repeatedly could explode stack size). In Tapscript (BIP-342), a 520-byte element size limit makes the memory concern manageable.

The opcode itself is simple: pop two stack items, push their concatenation. But combined with hash opcodes and OP_CHECKSIG, it enables surprisingly powerful constructions: merkle proof verification in script, STARK verifier contracts, tree signature schemes, vault-like covenants (by constructing transaction hash commitments), and sidechain bridge logic.

OP_CAT activates via the `OP_SUCCESS126` mechanism in Tapscript — opcodes that currently immediately succeed can be redefined by soft fork. The 520-byte element limit bounds worst-case stack growth.

BIP-347 has Status Complete, making it one of the most advanced covenant-adjacent proposals in the BIP process.
