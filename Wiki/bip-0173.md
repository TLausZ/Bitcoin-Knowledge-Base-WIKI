# BIP-0173: Base32 address format for native v0-16 witness outputs

## Summary

This document proposes "Bech32", a checksummed base32 format, and a standard for native segregated witness output addresses using it.

## Body

### Zweck

BIP-0173 defines bech32 — the address format used for native SegWit addresses, recognisable by their `bc1q...` prefix.

Bech32 uses a character set of 32 lowercase letters and digits, chosen to avoid visually ambiguous characters (no `0`/`O` or `I`/`l` confusion). Addresses are entirely lowercase, easier to read aloud, and less prone to transcription errors. The checksum can detect any single character error and locate the position of up to 4 errors — substantially stronger than Base58Check's simple CRC.

A bech32 address consists of a human-readable part (HRP), a separator `1`, and a data part encoding a witness version byte and the witness program. For Bitcoin mainnet the HRP is `bc`; for testnet it's `tb`. P2WPKH encodes a 20-byte key hash; P2WSH encodes a 32-byte script hash.

BIP-173 covers witness version 0 only. BIP-350 introduced bech32m — a modified checksum — for witness version 1 and above (Taproot `bc1p...`), after a subtle insertion weakness was found in the original bech32 checksum for 32-byte programs.
