# BIP-0012: OP_EVAL

## Summary

This BIP describes a new opcode (OP_EVAL) for the Bitcoin scripting system, and a new 'standard' transaction type that uses it to enable the receiver of bitcoins to specify the transaction type needed to re-spend them.

### Zweck

BIP-0012 proposed `OP_EVAL`, an opcode that would pop a script from the stack and execute it. This would allow a sender to pay to the hash of a script, with the actual script revealed only at spend time — the same goal later achieved by P2SH (BIP-16).

The proposal was rejected due to security concerns: recursive script evaluation could create cycles or deeply nested execution, making it difficult to reason about script validity and resource bounds. The script interpreter would need to handle an unbounded recursion depth. BIP-16 achieved the same goal with a much simpler, non-recursive mechanism — it just checks that the top stack element hashes to the embedded commitment, without evaluating scripts mid-execution.
