# BIP-0061: Reject P2P message

## Summary

This BIP describes a new message type for the Bitcoin peer-to-peer network.

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Gavin Andresen  
**Veröffentlicht:** 2014-06-18  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0061.mediawiki

### Zweck

BIP-0061 added a `reject` message to the Bitcoin P2P protocol that a node sends when it rejects a transaction or block from a peer. Before this, if a node rejected a transaction it simply didn't relay it — the sender had no way to know why the transaction was dropped.

The `reject` message includes a rejection code (e.g. `REJECT_INVALID`, `REJECT_DUST`) and an optional human-readable reason string. This helps wallet developers diagnose why transactions fail to propagate, and helps node operators debug policy mismatches between implementations.

However, the `reject` message is not trustworthy: a malicious peer could send false rejection messages to mislead clients about transaction validity. Bitcoin Core later disabled it by default (and removed support in version 21.0) after it was found to be more useful to attackers than to legitimate debugging. For reliable transaction status, querying a node's mempool directly is preferred.
