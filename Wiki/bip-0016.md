# BIP-0016: Pay to Script Hash

## Summary

This BIP describes a new "standard" transaction type for the Bitcoin scripting system, and defines additional validation rules that apply only to the new transactions.

### Zweck

P2SH (Pay-to-Script-Hash) is one of Bitcoin's most consequential consensus changes, activated in April 2012. It solved a fundamental problem with complex scripts: before P2SH, if Alice wanted to receive funds spendable by a 2-of-3 multisig, she had to give all three public keys to the sender, who then had to include them in the output. This made complex scripts expensive and burdensome for senders.

With P2SH, Alice instead provides only the hash of her redeem script. The sender pays to that hash (a 23-byte output). When Alice spends, she provides the actual redeem script plus the inputs to satisfy it. Validation: nodes check that the serialised script hashes to the embedded commitment, then execute the script with the provided inputs.

This moved script complexity from the sender to the recipient — the sender needs zero knowledge of Alice's spending conditions. It also moved the cost: the redeem script is revealed at spend time, not at output creation, so the UTXO set stays small.

P2SH enabled practical multisig wallets, hardware wallet recovery scripts, and was later extended by P2WSH (SegWit version of P2SH). Nearly every complex Bitcoin transaction today uses P2SH or its descendants.
