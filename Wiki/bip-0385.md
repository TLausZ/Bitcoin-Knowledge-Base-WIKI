# BIP-0385: raw() and addr() Output Script Descriptors

## Summary

This document specifies `raw()` and `addr()` output script descriptors. `raw()` encapsulates a raw script; `addr()` encapsulates an address.

## Body

**Typ:** Informational  
**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)  
**Themen:** bips, self-custody, privacy
**Autor(en):** Pieter Wuille, Ava Chow  
**Veröffentlicht:** 2021-06-27

### Zweck

BIP-385 defines two escape-hatch descriptors:

- `raw(SCRIPT_HEX)` — wraps an arbitrary hex-encoded script verbatim. Used to watch or interact with non-standard outputs that don't fit any standard descriptor function. The wallet can watch the output but may not be able to spend it automatically.
- `addr(ADDRESS)` — wraps a standard Bitcoin address. Converts the address back to its output script and watches for it. Useful for watch-only wallets that want to track an address they don't control the keys for.

Both are intentionally limited to watch-only use in most wallet implementations — `raw()` doesn't provide spending logic, and `addr()` requires separately managing the private key. They're primarily useful in watch-only wallet configurations and for monitoring third-party addresses.
