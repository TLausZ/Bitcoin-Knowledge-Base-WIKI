# BIP-0341: Taproot: SegWit version 1 spending rules

## Summary

This document proposes a new SegWit version 1 output type with spending rules based on Taproot, Schnorr signatures, and Merkle branches.

## Body

**Layer:** Consensus (soft fork)

### Zweck

Taproot (activated November 2021, block 709,632) is the most significant Bitcoin upgrade since SegWit. It introduces P2TR — Pay-to-Taproot — as SegWit witness version 1, recognisable by `bc1p...` addresses.

A P2TR output commits to a Taproot output key Q = P + t·G, where P is an internal public key and t is a hash of a Merkle root of script spending conditions. To spend, the payer either provides a Schnorr signature for Q directly (key-path), or reveals the internal key P, the specific script being used, and a Merkle proof that the script is committed in Q (script-path).

The privacy gain comes from the key-path: complex spending conditions (timelocked recovery, multisig) are hidden in the Merkle tree. When the happy-path is used, no scripts are revealed — the on-chain footprint is a single signature and single pubkey. Only when unusual conditions are invoked does the script structure appear. A 3-of-3 MuSig2 multisig via key-path is on-chain indistinguishable from single-sig.

BIP-341 uses the SegWit v1 version byte, with Tapscript (BIP-342) defining script rules for v1. The leaf version byte in the script tree allows future script upgrades to use new versions without another consensus change.
