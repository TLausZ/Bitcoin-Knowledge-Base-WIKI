# BIP-0102: Block size increase to 2MB

## Summary

Simple, one-time increase in total amount of transaction data permitted in a block from 1MB to 2MB.

## Body

**Typ:** Specification  
**Layer:** Consensus (hard fork)  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Autor(en):** Jeff Garzik  
**Veröffentlicht:** 2015-06-23  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0102.mediawiki

### Zweck

BIP-0102 was the most minimal block size increase proposal of the 2015 debate: a simple, one-time doubling from 1MB to 2MB via hard fork, with no further scheduled increases. Garzik pitched it as a pragmatic compromise — modest enough to avoid serious centralisation concerns, large enough to buy time while longer-term solutions were developed.

It attracted some support precisely because of its simplicity, but suffered the same fate as all hard fork block size proposals: insufficient community consensus to activate. By mid-2017, the debate ended when SegWit activated and the "New York Agreement" hard fork (SegWit2x, which included a 2MB block size increase) collapsed.

## Open Questions

- Would a 2MB hard fork have defused the block size civil war, or was the conflict primarily ideological rather than about the specific number?
