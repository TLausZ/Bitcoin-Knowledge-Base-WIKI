# BIP-0144: Segregated Witness (Peer Services)

## Summary

This BIP defines new messages and serialization formats for propagation of transactions and blocks committing to segregated witness structures.

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Eric Lombrozo, Pieter Wuille  
**Veröffentlicht:** 2016-01-08  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0144.mediawiki

### Zweck

BIP-0141 defines the *consensus* rules for SegWit. BIP-0144 defines how SegWit transactions and blocks are transmitted over the P2P network.

BIP-0144 adds a new transaction serialisation format that includes witness data. The format is distinguished from the legacy format by a two-byte marker (`0x00 0x01`) after the version field. Nodes that don't understand SegWit will parse these bytes incorrectly, so the extended format is only used between SegWit-capable peers. Backwards-compatible peers communicate using the legacy format (which strips the witness).

BIP-144 also defines the `NODE_WITNESS` service bit so peers can advertise SegWit support, and specifies how witness data is included when blocks are requested. Without BIP-144, SegWit's consensus changes would have no way to propagate through the network.
