# BIP-0122: URI scheme for Blockchain references / exploration

## Summary

This BIP proposes a URI scheme for looking up blocks, transactions, and addresses on a blockchain explorer.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Draft (noch im Entwurfsstadium)  
**Themen:** bips
**Autor(en):** Marco Pontello  
**Veröffentlicht:** 2015-08-29  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0122.mediawiki

### Zweck

BIP-0122 proposes a `blockchain:` URI scheme for deep-linking to specific blockchain data — blocks, transactions, or addresses — in a chain-agnostic way. The format includes a genesis block hash prefix to identify the chain: `blockchain:/<genesis-hash-prefix>/tx/<txid>` would open a transaction in a viewer, with the genesis hash distinguishing Bitcoin mainnet from testnet or other chains.

The proposal is useful for applications that want to link to on-chain data without hardcoding a specific block explorer URL. Rather than linking to `https://mempool.space/tx/<txid>`, an app could use the `blockchain:` URI and let the user's OS route it to their preferred explorer.

Remains in Draft; the scheme has not been widely registered with OS-level URI handlers.
