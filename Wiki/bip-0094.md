# BIP-0094: Testnet 4

## Summary

A new test network intended to replace Testnet 3, with consensus rule improvements that make it impractical to attack using only CPU mining.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Fabian Jahr  
**Veröffentlicht:** 2024-05-27  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0094.mediawiki

### Zweck

Testnet 3 (active since 2012) had become problematic by 2024. A known attack allowed adversaries to exploit the testnet's difficulty reset rule: if no block was found in 20 minutes, the next block could be mined at difficulty 1, letting an attacker rapidly generate thousands of blocks with CPU hardware and disrupt testing infrastructure for everyone.

BIP-0094 defines Testnet 4 with a tightened difficulty rule: the minimum difficulty exception only applies if the block timestamp is more than 20 minutes after the previous block's timestamp, *and* the block was mined with difficulty 1. The attacker's exploit of mining difficulty-1 blocks to spam the chain is closed off.

Testnet 4 launched with Bitcoin Core 27.0 in 2024. It has the same coin supply structure as mainnet but coins have no monetary value. Developers and wallet teams use it for testing without risking real funds.
