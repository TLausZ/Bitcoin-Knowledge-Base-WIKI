# BIP-0086: Key Derivation for Single Key P2TR Outputs

## Summary

This document suggests a derivation scheme for HD wallets whose keys are involved in single key P2TR (BIP 341) outputs as the Taproot internal key.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)

### Zweck

BIP-0086 defines the derivation path for single-key Taproot addresses: `m/86'/coin_type'/account'/change/index`. Taproot addresses (bech32m, starting with `bc1p`) use a tweaked public key as the output key. The `86'` purpose field identifies this address type to a restoring wallet.

BIP-86 specifies that wallets must apply the BIP-341 "key-only" tweak (tweaking the key with the hash of itself as the Merkle root) even for single-key spends. This ensures that a BIP-86 key cannot be accidentally treated as a Taproot key with a hidden script path, which would be a security risk. The tweak also means the derived child key used for signing is different from the raw BIP-32 derived key.

BIP-86 is increasingly the default for new wallet setups. Taproot single-key outputs are indistinguishable on-chain from Taproot outputs with complex script trees, providing inherent privacy: a single-key spend looks identical to a threshold multisig or HTLC spend.
