# BIP-0030: Duplicate transactions

## Summary

This document gives a specification for dealing with duplicate transactions in the block chain, in an attempt to solve certain problems the reference implementation has with them.

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Pieter Wuille  
**Veröffentlicht:** 2012-02-22  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0030.mediawiki

### Zweck

BIP-0030 fixed a vulnerability that allowed a transaction's txid to collide with an earlier unspent transaction. Because Bitcoin's UTXO set was keyed by txid, a new transaction with the same txid as an existing unspent output would overwrite it in the database. An attacker could craft a coinbase transaction (coinbase transactions have no inputs, so they're easiest to duplicate) that matched a previous one, wiping the earlier UTXO.

This wasn't theoretical: two pairs of duplicate coinbase transactions existed in the real blockchain before the fix (in blocks 91812/91722 and 91842/91880). BIP-30 made it consensus-invalid to include a transaction in a block if that txid already exists among unspent outputs. BIP-34 later made this rule redundant for coinbase transactions specifically, by requiring block height in the coinbase script — making every coinbase txid unique by construction.
