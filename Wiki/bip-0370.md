# BIP-0370: PSBT Version 2

## Summary

This document proposes a second version of the Partially Signed Bitcoin Transaction format (BIP-174) that allows inputs and outputs to be added to the PSBT after creation.

### Zweck

PSBTv0 (BIP-174) treats the transaction as fixed once created — the unsigned transaction is included in the global section, and inputs/outputs cannot be added afterward. This makes PSBTv0 unsuitable for multi-party transaction construction workflows (like CoinJoin) where participants add their inputs and outputs independently.

PSBTv2 removes the global unsigned transaction entirely and replaces it with individual input/output maps that can be constructed incrementally. Each input and output is its own independent record. A Creator starts the PSBT with some inputs/outputs; additional parties can add their own. A Combiner merges PSBTs from different participants. Once all parties have added their contributions, a Finaliser assembles the complete transaction.

This makes PSBTv2 the right format for collaborative transaction building: CoinJoin coordinators, payment splitters, payjoin (BIP-78), and channel opens. BIP-371 (Taproot PSBT fields) and BIP-373 (MuSig2 PSBT fields) both target PSBTv2 as well as PSBTv0.
