# BIP-0339: WTXID-based transaction relay

## Summary

This BIP describes two changes to the P2P protocol to support transaction relay based on the witness txid (wtxid) rather than the txid.

**Layer:** Peer Services

### Zweck

Before BIP-339, transactions were announced over P2P by their txid — the hash of the transaction excluding witness data. SegWit introduced the wtxid (witness txid), which hashes the full transaction including witness. For SegWit transactions, the txid and wtxid differ.

The problem: if two nodes are announced the same transaction by different peers using different witness data (a valid scenario for SegWit transactions where the witness can be modified without changing the txid), the receiving node has no way to track that they're the same underlying transaction. This could cause a node to download the same transaction multiple times, or miss that a transaction it already has has been re-announced with modified witness.

BIP-339 adds a `wtxidrelay` message to negotiate wtxid-based announcement. Nodes that support it exchange transaction announcements using wtxids instead of txids. This makes transaction tracking unambiguous and is also necessary groundwork for package relay (BIP-331).
