# BIP-0008: Version bits with lock-in by height

## Summary

This document specifies an alternative to BIP9 that corrects for a number of perceived mistakes. Block heights are used for start and timeout rather than POSIX timestamps. It additionally introduces an activation parameter that can guarantee activation of backward-compatible changes (further called "soft forks").

### Zweck

BIP-0008 is a revised soft fork activation mechanism, designed to fix two problems with BIP-9. First, it replaces POSIX timestamps with block heights for signalling windows — heights are deterministic and cannot be manipulated by miner timestamp variance. Second, it introduces the `LOT` (lockinontimeout) parameter. When `LOT=true`, nodes will force a soft fork to lock in at the end of the signalling period regardless of miner hash rate support, giving users a mechanism to activate changes even if miners refuse to signal.

The `LOT` parameter proved controversial: `LOT=true` nodes essentially issue an ultimatum to miners, while `LOT=false` nodes behave like BIP-9. Running different `LOT` values risks a chain split if activation is contested. During the 2021 Taproot activation, a modified version called Speedy Trial was used instead — a compromise that used BIP-8/BIP-9 hybrid logic with a short signalling window and `LOT=false`, followed by a flag-day lock-in if 90% threshold was met.
