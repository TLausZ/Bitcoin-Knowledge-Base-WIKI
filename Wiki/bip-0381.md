# BIP-0381: Non-Segwit Output Script Descriptors

## Summary

This document specifies `pk()`, `pkh()`, and `sh()` output script descriptors for legacy (non-SegWit) output types.

## Body

**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)

**Veröffentlicht:** 2021-06-27

### Zweck

BIP-381 defines the three legacy output script descriptor functions:

- `pk(KEY)` — Pay-to-PubKey (P2PK): outputs that push the full public key and require a signature. Satoshi's original output type, now rarely used.
- `pkh(KEY)` — Pay-to-PubKey-Hash (P2PKH): legacy `1...` addresses. The standard output type until SegWit.
- `sh(SCRIPT)` — Pay-to-Script-Hash (P2SH): `3...` addresses. Wraps any script in a hash commitment; spender reveals the script at spend time.

`sh()` can wrap multisig scripts, P2WPKH (creating P2SH-P2WPKH), or any other valid script, enabling nested descriptors like `sh(wpkh(KEY))` for the wrapped SegWit format used by many older wallets.
