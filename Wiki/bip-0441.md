# BIP-0441: Restoration of disabled script (Tapleaf 0xC2)

## Summary

This BIP introduces a new tapleaf version (0xc2) that restores Bitcoin script to pre-0.3.1 capabilities: re-enables disabled opcodes, increases element size limits, and removes several historical restrictions.

### Zweck

In 2010, Satoshi disabled a set of opcodes (OP_CAT, OP_SUBSTR, OP_LEFT, OP_RIGHT, OP_INVERT, OP_AND, OP_OR, OP_XOR, OP_2MUL, OP_2DIV, OP_MUL, OP_DIV, OP_MOD, OP_LSHIFT, OP_RSHIFT) after a vulnerability (CVE-2010-5137) showed they could be exploited to cause excessive computation. The current 520-byte stack element limit and 32-bit numeric restriction were also imposed at this time.

BIP-441 restores all these opcodes under a new Tapscript leaf version (0xc2), relying on BIP-440's varops budget to bound computational cost safely. It also relaxes limits: maximum stack element size rises to 4MB, total stack memory to 8MB, maximum stack objects to 32,768, and numerics become arbitrary-precision unsigned integers.

This would dramatically expand Bitcoin script's expressiveness — enabling hash functions on large data, arbitrary arithmetic, bitwise logic, and string operations. The tapleaf versioning means existing scripts are unaffected; 0xc2 is a new execution environment, not a change to current rules.
