# BIP-0199: Hashed Time-Locked Contract transactions

## Summary

This BIP describes a standard script for HTLCs, used for generalized off-chain contract negotiation.

### Zweck

A Hashed Time-Locked Contract (HTLC) is the cryptographic building block that makes atomic swaps and the Lightning Network possible. The script encodes two spending conditions: the recipient can claim the funds by revealing a preimage R such that SHA256(R) equals a known hash H (the "hashlock"), or the sender can reclaim the funds after a timeout (the "timelock").

This two-branch construction enables trustless cross-chain swaps: Alice locks BTC with hashlock H on Bitcoin; Bob locks ETH with the same H on Ethereum. If Bob reveals R to claim the ETH, Alice learns R and can claim the BTC. Neither party can cheat — if Bob doesn't reveal R, Alice gets her BTC back after the timeout.

The Lightning Network uses HTLCs at every hop in a payment path. BIP-199 was closed because it codified a script template that was already widely deployed, and subsequent work (Taproot, PTLCs) has moved toward point-time-locked contracts as a better alternative.
