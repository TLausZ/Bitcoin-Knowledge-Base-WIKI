# BIP-0124: Hierarchical Deterministic Script Templates

## Summary

This BIP defines a script template format that can be used by wallets to deterministically generate scripts with specific authorization policies using BIP-32 key derivation.

## Body

**Typ:** Informational  
**Layer:** Applications  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Themen:** bips, self-custody
**Autor(en):** Eric Lombrozo, William Swanson  
**Veröffentlicht:** 2015-11-20  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0124.mediawiki

### Zweck

BIP-0124 proposed a template language for describing Bitcoin scripts that include BIP-32 key derivation paths. The idea was to let wallets express policies like "2-of-3 multisig using key at m/44'/0'/0'/0/0, m/44'/0'/1'/0/0, and m/44'/0'/2'/0/0" in a portable format that any wallet could reconstruct.

The proposal was essentially an early descriptor format — it recognised that knowing "which keys to use" and "how they're derived" together constitute a complete wallet policy specification. It was closed, but the same need it identified was later met by Bitcoin Core's output descriptors (BIP-379 and related), which became the standard way to express wallet policies in a portable format.
