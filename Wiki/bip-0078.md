# BIP-0078: A Simple Payjoin Proposal

## Summary

This document proposes a protocol for two parties to negotiate a coinjoin transaction during a payment between them.

### Zweck

BIP-0078 (Payjoin, also called P2EP — Pay-to-Endpoint) is a privacy protocol where a sender and receiver collaborate to construct a transaction that includes inputs from both parties. On-chain, this looks like a normal payment, but it breaks the most common blockchain analysis heuristic: the "common input ownership" assumption (that all inputs in a transaction belong to the same wallet).

The flow: the sender constructs a PSBT with their inputs and sends it to the receiver's HTTP endpoint. The receiver adds one of their own inputs and possibly adjusts the change output, then returns the modified PSBT. The sender finalises and broadcasts. Neither party reveals private keys to the other.

The result: blockchain analysis can no longer reliably determine the payment amount or which outputs are change. This breaks chain surveillance tools that rely on input clustering.

BIP-78 requires a synchronous HTTP connection (the receiver must be running a Payjoin endpoint at payment time), which limits adoption to always-online merchants. BIP-77 (Async Payjoin) addresses this limitation. BTCPay Server has had BIP-78 support since 2020.
