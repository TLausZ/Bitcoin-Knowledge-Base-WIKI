# BIP-0390: musig() Descriptor Key Expression

## Summary

This document specifies a `musig()` key expression for output script descriptors, taking multiple keys and producing a MuSig2 aggregate public key.

## Body

**Typ:** Informational  
**Layer:** Applications  
**Status:** Draft (noch im Entwurfsstadium)  
**Autor(en):** Ava Chow

### Zweck

BIP-390 adds `musig(KEY1, KEY2, ..., KEYn)` as a key expression in the descriptor language. This takes multiple individual public keys and aggregates them into a single MuSig2 key (per BIP-327). The result can be used in any context that accepts a key — most importantly inside `tr()`.

Example: `tr(musig(xpub1, xpub2, xpub3))` — a Taproot output with a key-path that requires all 3 parties to sign via MuSig2. On-chain, this is indistinguishable from a single-key Taproot output.

Combined with `tr()` and `multi_a()`, it enables hybrid setups: `tr(musig(key1, key2), multi_a(2, key1, key2, recovery_key))` — normal spending via MuSig2 key-path (single signature), with a 2-of-3 Tapscript path as fallback. BIP-328 provides the key derivation; BIP-373 provides the PSBT fields; BIP-390 closes the loop by integrating the aggregate key into the descriptor language.
