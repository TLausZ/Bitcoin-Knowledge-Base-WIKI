# BIP-0156: Dandelion - Privacy Enhancing Routing

## Summary

Dandelion is a transaction routing mechanism providing formal anonymity guarantees. Transactions travel along a randomly selected path during the "stem phase" before being diffused normally during the "fluff phase."

### Zweck

When a user broadcasts a transaction, the originating node is often identifiable: the first node to send a `inv` for the transaction is likely the originator. Surveillance nodes that connect to many peers can map which node first announced a transaction and correlate it with the sender's IP address.

Dandelion splits propagation into two phases. In the "stem phase," the transaction is forwarded to a single randomly selected peer (not broadcast to all peers). That peer forwards it to another single peer, and so on along a random chain for a geometrically distributed number of hops (expected ~10). Then the transaction enters the "fluff phase" and is broadcast normally. An attacker monitoring the network sees the first broadcast but not the earlier single-hop forwards, making the originator much harder to identify.

BIP-156 was closed — the implementation was complex and the anonymity guarantees depended on having a critical mass of participating nodes. The Dandelion++ variant (from later research) addressed some limitations, but neither made it into Bitcoin Core as a BIP. The idea influenced thinking about privacy-preserving transaction propagation.
