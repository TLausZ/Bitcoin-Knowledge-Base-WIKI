# BIP-0064: getutxo message

## Summary

This document describes a small P2P protocol extension that performs UTXO lookups given a set of outpoints.

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Themen:** bips
**Autor(en):** Mike Hearn  
**Veröffentlicht:** 2014-06-10  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0064.mediawiki

### Zweck

BIP-0064 proposed a `getutxo` P2P message allowing a node to ask a peer whether specific UTXOs exist and what their values are. This would let SPV clients verify that specific coins are unspent without downloading the entire UTXO set.

The proposal was closed primarily due to DoS concerns: a malicious node could send thousands of `getutxo` queries, forcing honest nodes to perform expensive UTXO lookups. Unlike transaction queries (which are bounded by the mempool), UTXO queries could hit any output in the multi-GB UTXO set. There was also a privacy concern: `getutxo` queries reveal which UTXOs a client is interested in.

The need for lightweight UTXO verification has since been partially addressed by Utreexo (cryptographic UTXO set commitments) and by Electrum server infrastructure, neither of which requires changes to the Bitcoin P2P protocol.
