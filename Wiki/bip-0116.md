# BIP-0116: MERKLEBRANCHVERIFY

## Summary

This BIP proposes MERKLEBRANCHVERIFY, a new opcode which allows script writers to commit to a set of data elements and have one or more provided at redemption without revealing the entire set.

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)  
**Status:** Draft (noch im Entwurfsstadium)  
**Autor(en):** Mark Friedenbach, Kalle Alm, BtcDrak  
**Veröffentlicht:** 2017-08-25

### Zweck

BIP-0116 proposes `OP_MERKLEBRANCHVERIFY` (MBV), which verifies that a given data element is part of a committed Merkle tree. A script can commit to a root hash; the spender provides an element and a Merkle proof, and MBV checks that the proof is valid.

This is a building block for general MAST-like constructions without requiring changes to the overall script structure. Combined with BIP-117 (tail call semantics), a script can commit to multiple spending conditions in a Merkle tree and reveal only the executed condition at spend time — similar to Taproot's script tree but implemented at the script level rather than the key level.

Both BIP-116 and BIP-117 were rendered largely redundant by Taproot (BIP-341), which builds in MAST natively and more efficiently. They remain in Draft with no active push for activation.
