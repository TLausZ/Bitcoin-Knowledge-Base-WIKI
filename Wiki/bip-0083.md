# BIP-0083: Dynamic Hierarchical Deterministic Key Trees

## Summary

This BIP defines a scheme for key derivation that allows for dynamic creation of key hierarchies based on BIP-32.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Themen:** bips, self-custody
**Autor(en):** Eric Lombrozo  
**Veröffentlicht:** 2015-11-16  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0083.mediawiki

### Zweck

BIP-0083 proposed a way to derive HD key trees dynamically, where the path structure itself could be encoded as data and passed between parties rather than being fixed by convention. The idea was to allow applications to specify arbitrary tree structures without needing a BIP for each possible layout.

The proposal was closed without adoption. The community converged on the opposite approach: fixed, well-known path structures defined by individual BIPs (BIP-44, BIP-49, BIP-84, BIP-86, etc.) and path templates (BIP-88) for describing those fixed structures. Dynamic path generation introduces uncertainty at restore time — a wallet can't know what paths were used without the original application logic.
