# BIP-0374: Discrete Log Equality Proofs

## Summary

This document proposes a standard for 64-byte zero-knowledge discrete log equality proofs (DLEQ proofs) over an elliptic curve, proving that the same scalar is used in two different EC point multiplications.

### Zweck

A Discrete Log Equality Proof (DLEQ) proves that for points A, B, C, G on an elliptic curve: A = a·G and C = a·B, using the same scalar a — without revealing a. This is a non-interactive Schnorr-style zero-knowledge proof.

In Bitcoin, Silent Payments (BIP-352) are the main use case. When a sender creates a Silent Payment, they compute an ECDH shared secret between their input key and the receiver's scan key. If the sender uses multiple inputs, a DLEQ proof allows the receiver to verify that the shared secret was computed correctly from the declared inputs — without the sender revealing their private keys. A light client outsourcing scanning can rely on this: the scanning server can verify the DLEQ proof and confirm the sender computed the output key honestly.

The BIP standardises the proof format (64 bytes, structured like a Schnorr signature) and the hash construction, ensuring different implementations produce compatible proofs. BIP-374 proofs appear in BIP-375 (Silent Payments PSBT fields) for the same reason.
