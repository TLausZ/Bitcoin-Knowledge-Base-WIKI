# BIP-0322: Generic Signed Message Format

## Summary

A standard for interoperable signed messages based on Bitcoin Script, either for proving funds availability or for committing to a message as the intended recipient of funds.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Complete (abgeschlossen)

### Zweck

BIP-0137 defined message signing for legacy P2PKH addresses, but it doesn't extend cleanly to P2SH, P2WPKH, P2WSH, or Taproot. BIP-322 proposes a general scheme: to sign a message for an address, create a virtual transaction spending a virtual output to that address, then provide a valid signature for it. The signed message is essentially "I can spend funds at this address."

This works for any address type because it uses the same script validation engine as actual Bitcoin transactions. For P2WPKH, the signer produces the same witness stack they would for a real spend. For P2SH multisig, all required keys sign. For Taproot, either key-path or script-path spending can be used.

BIP-322 also defines a "proof of funds" variant where the virtual transaction spends real UTXOs — proving the signer controls specific unspent outputs. This is more powerful than BIP-127 proof-of-reserves for individual wallet users.

Despite Complete status, adoption has been slower than hoped — most wallets still implement the legacy BIP-137 format.
