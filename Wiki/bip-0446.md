# BIP-0446: OP_TEMPLATEHASH

## Summary

This BIP proposes OP_TEMPLATEHASH, an opcode that pushes a hash committing to specific fields of the spending transaction onto the stack.

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)  
**Status:** Draft (noch im Entwurfsstadium)

**Veröffentlicht:** 2026-02-06

### Zweck

OP_TEMPLATEHASH is an alternative to BIP-119 (OP_CTV) and BIP-346 (OP_TXHASH) for transaction introspection. It pushes a hash of selected transaction fields onto the stack, allowing scripts to verify that the spending transaction matches a required template.

Unlike OP_CTV, OP_TEMPLATEHASH commits to a user-specified combination of transaction fields via a template selector, rather than committing to a fixed full-transaction hash as BIP-119 does. This makes it more flexible while remaining simpler than OP_TXHASH's general field selector.

BIP-446 was developed by the same authors as BIP-448 (rebindable transactions) as part of an integrated approach to advanced transaction introspection and covenant capabilities. It's part of the active 2025-2026 covenant design space alongside OP_CTV, OP_TXHASH, and OP_CCV.
