# BIP-0440: Varops Budget For Script Runtime Constraint

## Summary

This BIP defines a "varops budget" generalizing the sigops budget from BIP-342 to all variable-cost operations, providing a unified resource accounting framework for future script opcodes.

### Zweck

Bitcoin script has various per-script resource limits: maximum stack depth, maximum element size, maximum opcode count, and a "sigops budget" (BIP-342) that counts signature operations. These limits prevent scripts from being too computationally expensive to validate, but they're piecemeal — each new opcode type that introduces variable cost needs its own limit.

BIP-440 proposes a unified "varops budget" that can account for any computationally variable operation in a single per-input budget. New opcodes like OP_CAT (concatenation is O(n)), hash operations, or any future opcode with cost proportional to data size can be charged against this budget rather than needing new ad-hoc limits.

This BIP is infrastructure for BIP-441 (restored disabled opcodes) — without a unified cost model, re-enabling opcodes like OP_CAT with relaxed size limits would be risky. With a varops budget, the cost is bounded regardless of element sizes.
