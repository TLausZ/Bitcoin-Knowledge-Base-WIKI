# BIP-0054: Consensus Cleanup

## Summary

This document proposes several small consensus rule fixes to address long-standing vulnerabilities and technical debt in the Bitcoin protocol.

### Zweck

BIP-0054 bundles several consensus-level fixes that are individually uncontroversial but collectively address real vulnerabilities.

The timewarp attack: Bitcoin's difficulty adjustment has a flaw where a miner controlling majority hashrate can manipulate timestamps to make retarget periods appear shorter than they are, enabling block production at artificially low difficulty — indefinitely. BIP-54 restricts how much the first block of a new period's timestamp can deviate from the last block of the previous period, closing this attack.

Legacy (non-SegWit) transaction signature hashing scales quadratically with input count: computing the sighash for each input requires re-serialising the entire transaction, so a transaction with N inputs requires O(N²) work. A specially crafted transaction can force validators to do gigabytes of hashing. BIP-54 caps legacy transaction size to bound this cost.

The proposal also includes a fix for 64-byte transactions, which can create ambiguity in the transaction Merkle tree (used to generate SPV proofs), and minor cleanup to coinbase handling. Together these close known attack vectors that have been documented for years but not yet fixed.
