# BIP-0433: Pay to Anchor (P2A)

## Summary

This document describes Pay to Anchor (P2A) — a new standard output type with no spending key requirement, designed for fee bumping via CPFP.

### Zweck

Pay-to-Anchor is a tiny, keyless output designed to be attached to pre-signed transactions (like Lightning commitment transactions) specifically so that the holder can spend it to attach a high-fee CPFP child. The output script is `OP_1 <2-byte witness program>` — essentially an "anyone can spend" SegWit v1 output. Anyone who holds the parent transaction can spend the anchor output with a CPFP transaction to fee-bump the parent.

The "keyless" property is intentional: P2A outputs are explicitly intended to be ephemeral fee bumping hooks, not value-bearing outputs. Their script has no signing requirement, so they can be spent by anyone — but they're typically spent by the party who created the parent transaction (since they're the ones who want it confirmed quickly).

Combined with TRUC (BIP-431), P2A anchors allow Lightning nodes to set pre-signed commitment transactions to zero fee and add fees at broadcast time via CPFP, solving the "stuck commitment transaction" problem without needing pre-committed fee levels.
