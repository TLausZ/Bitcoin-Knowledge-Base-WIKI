# BIP-0353: DNS Payment Instructions

## Summary

This BIP proposes a standard format for encoding BIP-21 URI schemes in DNS TXT records, enabling human-readable Bitcoin payment addresses.

## Body

**Layer:** Applications  
**Status:** Complete (abgeschlossen)

### Zweck

BIP-0353 lets domain owners publish Bitcoin (and Lightning) payment instructions in DNS TXT records. A wallet that supports BIP-353 can resolve `user@domain.com` to a full BIP-21 URI, which may contain an on-chain address, a Lightning BOLT12 offer, a Silent Payments address, or any combination.

The DNS record format uses a special subdomain: `₿user.domain.com` (or `user.user._bitcoin-payment.domain.com`). The TXT record contains the `bitcoin:` URI. Wallets query this via DNS (optionally over HTTPS for privacy, or via DNSSEC for security).

The practical result: instead of sharing a QR code or address string, a person can share their email address (or just a `name@domain` handle), and wallets resolve it to the actual payment instructions. The domain owner controls the DNS record, so payment instructions can be updated (e.g., rotating Lightning offers).

BIP-353 is already deployed in a number of wallets and Lightning clients, with support for both on-chain (via BIP-321) and Lightning (BOLT12) addresses.
