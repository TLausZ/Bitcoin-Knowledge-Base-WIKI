# BIP-0121: Proof of Payment URI scheme

## Summary

This is a proposal for a URI scheme to be used in the Proof of Payment process (BIP-120).

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Autor(en):** Kalle Rosenbaum  
**Veröffentlicht:** 2015-07-27  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0121.mediawiki

### Zweck

BIP-0121 defined the URI scheme for triggering a BIP-120 Proof of Payment flow. A merchant would present a URI like `btcpop:?p=<payment_url>&n=<nonce>`, which a wallet could scan to initiate the proof. The URI carries the endpoint where the wallet should post the PoP, and a nonce to prevent replay.

Closed alongside BIP-120 since neither the proof mechanism nor the URI scheme were broadly adopted.
