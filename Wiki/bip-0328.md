# BIP-0328: Derivation Scheme for MuSig2 Aggregate Keys

## Summary

This document specifies how BIP-32 extended public keys can be constructed from a BIP-327 MuSig2 aggregate public key and how such keys should be used for key derivation.

### Zweck

BIP-327 produces an aggregate public key from multiple signers' keys. To receive multiple payments to a MuSig2 wallet without reusing addresses, you need to derive child keys from the aggregate key — but BIP-32 derivation was defined for individual keys, not aggregates.

BIP-328 specifies how to treat the MuSig2 aggregate public key as the root of a BIP-32 key tree. The aggregate key itself becomes the xpub, and child keys are derived by tweaking it in the same way BIP-32 works for standard keys. Each cosigner independently derives the corresponding child private key from their own individual key, then signs using MuSig2.

This integration makes MuSig2 wallets compatible with existing BIP-32/44/86 derivation tooling and allows wallet descriptors to represent MuSig2 setups. BIP-390 defines the `musig()` descriptor expression that uses BIP-327 and BIP-328 together.
