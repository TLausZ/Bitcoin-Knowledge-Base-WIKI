# BIP-0127: Simple Proof-of-Reserves Transactions

## Summary

This BIP describes a simple way to construct proof-of-reserves transactions using regular Bitcoin transaction serialization and BIP-174 PSBT format.

### Zweck

BIP-0127 standardises how exchanges and custodians prove they control specific Bitcoin addresses without spending the funds. The mechanism: create a PSBT (BIP-174) transaction that spends all the addresses being proved, with a specially crafted "proof" input (a fake input spending a non-existent UTXO with the commitments message as scriptSig). This PSBT is then signed with the private keys of all addresses. The signatures can be verified by anyone without broadcasting — it's a provably valid set of signatures over the claimed UTXOs.

The standard allows auditors to verify that a custodian actually controls the keys for their claimed reserves. On its own it doesn't prove liabilities (you also need proof that those addresses correspond to customer balances), but it addresses the "do you actually control the coins?" question. Exchanges like Kraken have used this approach; the BIP formalises a format that was previously done ad-hoc.
