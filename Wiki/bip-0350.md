# BIP-0350: Bech32m format for v1+ witness addresses

## Summary

This document defines Bech32m, an improved variant of Bech32, and amends BIP-173 to use Bech32m for native segregated witness outputs of version 1 and later.

### Zweck

BIP-173's Bech32 has a subtle flaw: if the last character before the checksum is a `p`, inserting or removing `q` characters immediately before it doesn't change the checksum. This means certain address modifications go undetected. For 32-byte witness programs (like Taproot P2TR), this could theoretically cause funds to be sent to the wrong address after a specific pattern of typos.

BIP-350 fixes this by defining Bech32m: the same algorithm as Bech32 but with the checksum constant changed from `1` to `0x2bc830a3`. The modified constant eliminates the insertion weakness. Bech32 (original) remains in use for SegWit v0 addresses (P2WPKH `bc1q...` and P2WSH `bc1q...`). Bech32m is used for SegWit v1+ addresses, including Taproot (`bc1p...`).

The distinction matters for wallet developers: the right checksum algorithm must be applied based on the witness version. A wallet that applies bech32 to a Taproot address will accept valid-looking but incorrectly checksummed addresses.
