# BIP-0346: OP_TXHASH

## Summary

This BIP proposes OP_TXHASH, an opcode that pushes a hash of (user-specified fields of) the current transaction onto the stack.

### Zweck

OP_TXHASH enables Bitcoin scripts to inspect parts of the spending transaction. A script can push a hash of selected transaction fields (inputs, outputs, amounts, locktime, etc.) onto the stack, then verify it against an expected value. This enables "covenants" — spending conditions that constrain what the next transaction looks like.

The opcode takes a "txhash field selector" argument that specifies which transaction fields to include in the hash. Different field combinations allow scripts to commit to specific outputs, input amounts, or the complete transaction structure.

OP_TXHASH is more general than BIP-119 (CTV), which commits to a fixed template hash. OP_TXHASH allows flexible field selection, making it composable with other opcodes like CHECKSIGFROMSTACK (BIP-348). The pair (OP_TXHASH + CHECKSIGFROMSTACK) enables arbitrary transaction introspection without pre-committed templates.
