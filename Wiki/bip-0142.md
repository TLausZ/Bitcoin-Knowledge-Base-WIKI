# BIP-0142: Address Format for Segregated Witness

## Summary

This BIP describes new types of Bitcoin address to support native segregated witness transactions with 20-byte and 32-byte programs.

### Zweck

BIP-0142 was an early address format proposal for native SegWit outputs — it predates and was superseded by bech32 (BIP-173). It proposed using Base58Check encoding with new version bytes for P2WPKH and P2WSH addresses, similar to how P2SH addresses use version byte `0x05`.

The proposal was closed in favour of BIP-173 (bech32), which has several advantages: bech32 uses only lowercase letters and digits with no ambiguous characters (no 0/O or I/l confusion), includes stronger error detection that can locate the position of errors, and starts with a human-readable prefix (`bc1`) that makes address type immediately obvious to users.
