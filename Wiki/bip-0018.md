# BIP-0018: hashScriptCheck

## Summary

This BIP modifies the basic format of transaction inputs and outputs, replacing the current scriptSig and scriptPubKey with new contents: dataSig, scriptCheck, and hashScriptCheck.

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)  
**Status:** Complete (abgeschlossen)  
**Themen:** bips
**Autor(en):** Luke Dashjr

### Zweck

BIP-0018 was the most radical of the three competing P2SH proposals (alongside BIP-12/OP_EVAL and BIP-16/P2SH). Rather than adding an opcode or repurposing existing script behaviour, it proposed restructuring the entire transaction format: rename `scriptSig` to `dataSig`, rename `scriptPubKey` to `hashScriptCheck`, and introduce `scriptCheck` as the explicit script field in inputs.

The advantage of this approach was conceptual clarity — the transaction format would explicitly encode where the script lives, rather than relying on implicit conventions. The disadvantage was the scope of the change: restructuring the transaction format is a much larger consensus change than adding a new standard script type, and it would have required broader protocol-level updates.

It was not adopted. BIP-16 won because it achieved the same practical goal with a much smaller consensus footprint. BIP-18 is marked "Complete" in the sense that the proposal document was finalised, not that the proposal was deployed.
