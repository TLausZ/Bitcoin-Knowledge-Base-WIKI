# BIP-0038: Passphrase-protected private key

## Summary

A method is proposed for encrypting and encoding a passphrase-protected Bitcoin private key record in the form of a 58-character Base58Check-encoded printable string. Each record string contains all the information needed to reconstitute the private key except for a passphrase, and the methodology uses salting and scrypt to resist brute-force attacks.

### Zweck

BIP-0038 defines a standard for encrypting a Bitcoin private key with a passphrase, producing a 58-character Base58Check string that starts with `6P`. The encryption uses scrypt for key derivation (making brute-force expensive) and AES-256-CBC for the encryption itself. To recover the key, you need both the BIP-38 string and the passphrase.

Physical storage — paper wallets or metal backups where the private key is printed/engraved in encrypted form. If the physical medium is stolen, the attacker has the encrypted key but still needs to brute-force the passphrase. This is different from seed phrase backup (BIP-39) — BIP-38 encrypts a single private key, not an entire HD wallet seed.

The standard has also been used by physical Bitcoin manufacturers (commemorative coins with private keys under a hologram) to allow verification of the public key without requiring the passphrase.
