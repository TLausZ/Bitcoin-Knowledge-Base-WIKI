# BIP-0340: Schnorr Signatures for secp256k1

## Summary

This document proposes a standard for 64-byte Schnorr signatures over the elliptic curve secp256k1.

### Zweck

BIP-0340 specifies Schnorr signatures for Bitcoin using the same secp256k1 curve as ECDSA. Schnorr has several concrete advantages.

Signatures are linear: multiple signatures can be combined into one aggregate signature that looks identical to a single-key signature. Two parties each holding a key can sign together and produce output indistinguishable from a solo signature. This is the mathematical basis for MuSig2 (BIP-327) and Taproot key-path spending — a 3-of-3 multisig can spend with one 64-byte signature rather than three.

Schnorr's security also reduces to the discrete logarithm problem under standard assumptions. ECDSA's security properties are harder to formally reason about, requiring additional non-standard assumptions. Multiple Schnorr signatures can be batch-verified together faster than verifying each individually, reducing block validation time.

The format uses 64-byte signatures (32-byte R x-coordinate plus 32-byte s scalar) and x-only public keys (32 bytes, since the y-coordinate can be inferred). The hash function is tagged — messages are prefixed with a domain separator before hashing — to prevent cross-protocol signature reuse.

BIP-340 was activated as part of Taproot in November 2021 and is used in Taproot key-path and script-path spending.
