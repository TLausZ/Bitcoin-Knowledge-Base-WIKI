# BIP-0152: Compact Block Relay

## Summary

Compact block relay is a method of reducing the bandwidth needed to propagate new blocks, by taking advantage of the fact that nodes share a common mempool.

### Zweck

When a miner finds a block, propagating it to the rest of the network as fast as possible is economically important — other miners working on the old chain are wasting hashrate. Before compact blocks, full block propagation required transmitting all transaction data even when receiving nodes already had those transactions in their mempool.

BIP-0152 defines "compact blocks": a block announcement that includes only short 6-byte transaction identifiers (a truncated version of the txid) rather than full transactions. Receiving nodes reconstruct the full block from their mempool using these short IDs, requesting only the transactions they don't already have via `getblocktxn`. For a typical block where the recipient has all transactions in mempool already, this reduces bandwidth by ~99%.

BIP-152 defines two modes. "High bandwidth" mode: nodes send compact blocks immediately upon receiving a block, without waiting for validation (reducing latency at the cost of potentially relaying invalid blocks). "Low bandwidth" mode: send only after validation. Most well-connected nodes use high bandwidth mode for a subset of peers.
