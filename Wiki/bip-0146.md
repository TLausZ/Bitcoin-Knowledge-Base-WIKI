# BIP-0146: Dealing with signature encoding malleability

## Summary

This document specifies proposed changes to fix signature malleability related to ECDSA signature encoding.

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Autor(en):** Johnson Lau, Pieter Wuille  
**Veröffentlicht:** 2016-08-16  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0146.mediawiki

### Zweck

BIP-0146 addressed remaining malleability vectors not covered by BIP-66 (strict DER) and BIP-147 (dummy stack element). Specifically: the ECDSA S-value in a signature can be either of two values (S or n-S where n is the curve order) and both are valid. Normalising signatures to always use the low-S value eliminates this source of malleability.

The proposal also covered additional edge cases in `OP_CHECKMULTISIG` encoding. It was eventually closed because SegWit (BIP-141) addressed malleability more comprehensively for SegWit inputs: since SegWit witness data isn't included in the txid hash, malleating it doesn't change the txid. For legacy inputs, the remaining malleability concerns are now considered acceptable. BIP-146 was not worth activating as a standalone change after SegWit.
