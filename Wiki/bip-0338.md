# BIP-0338: Disable transaction relay message

## Summary

This BIP describes a P2P change allowing a node to tell a peer that a connection will not be used for transaction relay, supporting block-relay-only connections.

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Themen:** bips
**Autor(en):** Suhas Daftuar  
**Veröffentlicht:** 2020-09-03  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0338.mediawiki

### Zweck

Bitcoin Core maintains a subset of connections dedicated purely to block relay — no transaction relay. These connections improve Eclipse attack resistance: an attacker who wants to isolate a node must control all of its block-relay peers, not just its transaction relay peers. More block-relay-only connections mean a harder attack.

The problem: there was no way for a node to signal "this connection is for blocks only" to its peer. The peer would attempt to relay transactions anyway, wasting bandwidth. BIP-338 defined a `disabletx` message to signal block-relay-only mode at connection time.

BIP-338 was closed — the functionality was implemented in Bitcoin Core before the BIP was formally completed, making the BIP an after-the-fact description of deployed behaviour rather than a forward-looking specification.

## Open Questions
