# BIP-0325: Signet

## Summary

A new type of test network where signatures are used in addition to proof of work for block progress, enabling reliable and coordinated testing for multiple independent parties.

### Zweck

Testnet has a chronic usability problem: because it uses real proof-of-work, hashrate can fluctuate wildly. When miners run high-powered hardware on testnet, blocks come in rapidly and the difficulty shoots up; when they leave, the difficulty stays high and blocks slow to a crawl. The difficulty reset rule (any block with a 20-minute gap resets to minimum difficulty) causes erratic block times.

Signet adds a signature requirement on top of proof-of-work: valid blocks must contain a signature from a trusted key. This makes the network controllable — whoever holds the signing key determines what blocks are valid. The default global signet (maintained by the community) produces blocks approximately every 10 minutes, reliably, making it practical for long-running tests involving multiple parties.

Anyone can run a custom signet with their own signing key, for private testing environments. This makes signet useful both for ecosystem testing (the global default) and private protocol development.

Signet is supported in Bitcoin Core, and most wallets that support testnet now also support signet.
