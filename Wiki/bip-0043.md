# BIP-0043: Purpose Field for Deterministic Wallets

## Summary

This BIP introduces a "Purpose Field" for use in deterministic wallets based on algorithm described in BIP-0032.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Marek Palatinus, Pavol Rusnak  
**Veröffentlicht:** 2014-04-24  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0043.mediawiki

### Zweck

BIP-0043 adds a "purpose" level as the first hardened derivation step in the BIP-32 path: `m/purpose'/...`. The purpose field identifies which BIP defines the rest of the path structure. Each BIP that defines a derivation path uses its own BIP number as the purpose value.

For example: BIP-44 uses purpose `44'`, BIP-49 uses `49'`, BIP-84 uses `84'`, BIP-86 uses `86'`. This means a wallet restoring from seed can inspect the purpose field and immediately know which address type and path structure to use — `m/44'/...` implies legacy P2PKH, `m/84'/...` implies native SegWit bech32.

Without this convention, wallets would need to scan all possible path structures on restore, or users would have to manually specify what type of addresses their wallet used. BIP-43 is a meta-standard: it defines only the first path level and delegates everything below it to separate BIPs.
