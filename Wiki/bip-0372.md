# BIP-0372: Pay-to-contract tweak fields for PSBT

## Summary

This document proposes additional PSBT fields for pay-to-contract (P2C) key tweaking data, allowing signers to produce valid signatures for tweaked keys.

## Body

**Layer:** Applications

### Zweck

Pay-to-contract (P2C) is a technique where a public key is tweaked by adding a contract hash commitment: Q = P + hash(P, contract)·G. The resulting key Q is the one that appears on-chain. A signer who knows both the original key P and the tweak can sign for Q.

In a PSBT workflow, the signer needs to know the tweak to produce a valid signature — but the tweak isn't part of the standard PSBT input fields. BIP-372 adds new PSBT fields to carry the contract hash and the original (pre-tweak) key, allowing hardware wallets and offline signers to reconstruct Q and sign correctly.

This is relevant for RGB protocol (a client-validated smart contracts layer on Bitcoin), Lightning channel commitments, and other systems that use key tweaking to commit to off-chain data. As a Draft, it has primarily seen interest from the RGB/Taproot ecosystem.
