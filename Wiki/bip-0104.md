# BIP-0104: 'Block75' - Max block size like difficulty

## Summary

Automatic adjustment of max block size with the target of keeping blocks 75% full, based on the average block size of the previous 2016 blocks.

## Body

**Typ:** Specification  
**Layer:** Consensus (hard fork)  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Autor(en):** t.khan  
**Veröffentlicht:** 2017-01-13  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0104.mediawiki

### Zweck

BIP-0104 ("Block75") proposed an algorithmic block size limit modelled on Bitcoin's difficulty adjustment. Every 2016 blocks, the limit would adjust to keep average block fullness at 75% — if blocks were consistently fuller than 75%, the limit would rise; if they were less full, it would fall. The adjustment would be capped at ±25% per period.

The appeal: no hard-coded schedule, no miner votes, just a market-driven feedback loop. The concern: 75% fullness as a target would permanently maintain a fee market where blocks are often full, which users might experience as consistently high fees. Also, like all block size proposals of this era, it required a hard fork. Closed without activation.
