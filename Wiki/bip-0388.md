# BIP-0388: Wallet Policies for Descriptor Wallets

## Summary

This document specifies wallet policies — a compact, reviewable format built on descriptors for representing wallet accounts on hardware signing devices.

### Zweck

Full output script descriptors can be verbose, especially for multisig setups with long xpubs. Hardware signing devices have small screens and limited memory — displaying a 200-character descriptor and asking users to verify it is impractical.

BIP-388 introduces wallet policies: a compact format that separates the descriptor template (the structural policy) from the key information (the xpubs). Instead of inlining full xpubs, the template uses placeholders like `@0`, `@1`, `@2` that reference a separate list of keys. A hardware wallet can display: "This is a 2-of-3 multisig between @Alice, @Bob, @Carol" and verify that the xpubs match those the user expects — without needing to show the full derivation paths on screen.

The format also allows "account" descriptors that cover both receive and change paths in a single policy (using `<0;1>` as a derivation step). BIP-388 is implemented in the Ledger Bitcoin app (v2.1.0+) and Liana wallet, and is the standard for hardware wallet multisig registration.
