# BIP-0079: Bustapay :: a practical coinjoin protocol

## Summary

Bustapay is a simple and practical protocol for the sender and receiver of a payment to collaboratively sign a bitcoin transaction in such a way that busts some blockchain analysis assumptions.

### Zweck

BIP-0079 (Bustapay) was an early Payjoin-like proposal that predates BIP-78. Like BIP-78, it proposed having the receiver add inputs to a sender's transaction to break the common-input-ownership heuristic. The protocol was simpler than BIP-78: the sender sends a signed transaction to a Bustapay endpoint; the receiver adds their input and a new output for change, then re-signs and broadcasts.

Bustapay was closed in favour of BIP-78, which refined the protocol significantly: BIP-78 uses PSBT instead of fully-signed transactions (allowing the receiver to modify the transaction more safely), defines more precise security properties, and handles edge cases (fee rates, output ordering, version compatibility) that Bustapay left unspecified.

BIP-79 is primarily of historical interest as the precursor that demonstrated the Payjoin concept was implementable before BIP-78 formalised it.
