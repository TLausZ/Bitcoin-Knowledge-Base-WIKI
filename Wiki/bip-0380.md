# BIP-0380: Output Script Descriptors General Operation

## Summary

This document describes the general syntax for output script descriptors, including descriptor checksums and common key expressions.

## Body

**Typ:** Informational  
**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)  
**Themen:** bips, self-custody, privacy

**Veröffentlicht:** 2021-06-27

### Zweck

Output script descriptors are a human-readable language for describing how to produce Bitcoin output scripts and the corresponding spending keys. Before descriptors, wallets used implicit conventions (BIP-32 derivation paths, xpubs) that required out-of-band knowledge to interpret correctly. Descriptors make a wallet's structure self-describing.

A descriptor like `wpkh([deadbeef/84'/0'/0']xpub.../0/*)` unambiguously says: derive keys from the xpub at depth 84'/0'/0' (fingerprint deadbeef), using path /0/*, and wrap each in P2WPKH. The wallet knows exactly which addresses to watch and how to sign. No implicit convention, no ambiguity.

BIP-380 defines the grammar shared by all descriptor types: key expressions (raw keys, xpub derivations, key origins), operator precedence, and the checksum algorithm (using the same BCH encoding as bech32). The specific output types are defined in BIPs 381–392.

Descriptors are now the standard wallet backup and import format in Bitcoin Core and are supported by Sparrow, Specter, Electrum, and most modern wallets.
