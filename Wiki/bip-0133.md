# BIP-0133: feefilter message

## Summary

Add a new message, "feefilter", which instructs peers not to send "inv"s to the node for transactions with fees below the specified fee rate.

## Body

**Layer:** Peer Services  
**Status:** Deployed (aktiviert und im Einsatz)

### Zweck

BIP-0133 lets nodes tell their peers "don't bother announcing transactions to me unless they pay at least X sat/vbyte." Without this, a node's peers would announce all mempool transactions via `inv` messages, and the node would need to request and download transactions it would immediately reject as too low-fee.

As Bitcoin's fee market developed and nodes started enforcing minimum fee rates, this became wasteful: nodes with a fee floor of 1 sat/vbyte were being spammed with announcements for 0-fee or near-zero transactions they'd never accept. The `feefilter` message lets a node broadcast its effective minimum fee rate at connection time, so peers skip announcing transactions below that threshold.

This reduces bandwidth and processing load for nodes with non-zero minimum relay fees, and also reduces unnecessary p2p traffic network-wide during high-fee periods.
