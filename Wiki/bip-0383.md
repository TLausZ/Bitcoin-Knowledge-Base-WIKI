# BIP-0383: Multisig Output Script Descriptors

## Summary

This document specifies `multi()` and `sortedmulti()` output script descriptors for threshold multisig output scripts.

## Body

**Typ:** Informational  
**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)  
**Themen:** bips, self-custody, privacy
**Autor(en):** Pieter Wuille, Ava Chow  
**Veröffentlicht:** 2021-06-27  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0383.mediawiki

### Zweck

BIP-383 defines multisig descriptor functions:

- `multi(k, KEY1, KEY2, ..., KEYn)` — produces an `OP_k KEY1 KEY2 ... KEYn OP_n OP_CHECKMULTISIG` script. Keys appear in the script in the order given in the descriptor.
- `sortedmulti(k, KEY1, KEY2, ..., KEYn)` — same, but sorts the derived public keys lexicographically before inserting them. This ensures the script is the same regardless of key order in the descriptor, which matters for wallet interoperability: Ledger and Coldcard may derive keys in different order, but `sortedmulti()` guarantees they produce the same address.

`sortedmulti()` is the standard choice for multi-vendor multisig setups. It's combined with `wsh()` for native SegWit multisig or `sh()` for P2SH multisig.
