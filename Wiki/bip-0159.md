# BIP-0159: NODE_NETWORK_LIMITED service bit

## Summary

Define a service bit that allows pruned peers to signal their limited block serving capabilities.

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Deployed (aktiviert und im Einsatz)  
**Themen:** bips
**Autor(en):** Jonas Schnelli  
**Veröffentlicht:** 2017-05-11  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0159.mediawiki

### Zweck

Pruned nodes keep the full UTXO set and all recent blocks (by default, the last 550 blocks), but delete older block data to save disk space. They can validate new blocks and transactions but cannot serve historical block data to peers.

Before BIP-159, pruned nodes would set `NODE_NETWORK=0`, which made them indistinguishable from non-archival nodes that happen to have no blocks at all. Clients needing a specific historical block had no way to know which peers could serve it.

BIP-159 adds the `NODE_NETWORK_LIMITED` service bit (value 1024). Pruned nodes set this bit to indicate "I have the last 288+ blocks but not full history." Clients can then query these nodes for recent block data without attempting to fetch historical blocks they don't have. This makes pruned nodes more useful members of the network while being transparent about their limitations.
