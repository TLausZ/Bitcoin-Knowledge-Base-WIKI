# BIP-0010: Multi-Sig Transaction Distribution

## Summary

The enabling of multi-signature transactions in Bitcoin will introduce a great deal of extra functionality to the users of the network, but also a great deal of extra complexity. Executing a multi-signature tx will be a multi-step process, and will potentially get worse with multiple clients.

### Zweck

BIP-0010 was an early attempt to define a standard data format for passing partially-signed multi-signature transactions between participants. In a 2-of-3 multisig, three parties need to coordinate signing without a shared server. Without a standard format, each wallet would implement its own serialisation, making cross-wallet cooperation impossible.

The proposal defined a text-based transaction distribution format that could be sent via email or any other channel. It was eventually abandoned because the approach was too narrowly scoped and the format didn't generalise well. The same problem is now solved by PSBT (BIP-174), which handles partially-signed transactions for any script type, not just multisig.
