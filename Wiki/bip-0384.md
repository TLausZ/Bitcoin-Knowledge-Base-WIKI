# BIP-0384: combo() Output Script Descriptors

## Summary

This document specifies `combo()` output script descriptors, which take a key and produce all applicable output scripts (P2PK, P2PKH, P2WPKH, P2SH-P2WPKH).

## Body

**Typ:** Informational  
**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Pieter Wuille, Ava Chow  
**Veröffentlicht:** 2021-06-27  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0384.mediawiki

### Zweck

`combo(KEY)` is a convenience descriptor for wallet recovery when the address type is unknown. Given a key, it generates all output scripts that could plausibly have been used: P2PK, P2PKH, P2WPKH (if the key is compressed), and P2SH-P2WPKH. The wallet watches all of them simultaneously.

This is primarily useful for importing a WIF private key or an xpub without knowing which address format was used. Rather than requiring the user to specify the address type, `combo()` covers all bases. It's not recommended for creating new wallets — use the specific descriptor for the intended address type. Its value is in recovery and import scenarios.
