# BIP-0084: Derivation scheme for P2WPKH based accounts

## Summary

This BIP defines the derivation scheme for HD wallets using the P2WPKH (BIP 173) serialization format for segregated witness transactions.

## Body

**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)

### Zweck

BIP-0084 defines the derivation path for native SegWit (bech32) addresses: `m/84'/coin_type'/account'/change/index`. The `84'` purpose field tells a wallet restoring from seed to generate P2WPKH outputs (addresses starting with `bc1q`) rather than legacy P2PKH (`1...`) or wrapped SegWit P2SH-P2WPKH (`3...`).

This distinction is critical at restore time. If a user set up a native SegWit wallet and their wallet software uses `m/84'`, a wallet that only scans `m/44'` on restore will find nothing and incorrectly report zero balance. The purpose field in the path is what makes address-type-aware scanning possible.

Native SegWit (P2WPKH) is preferred over wrapped SegWit (BIP-49/P2SH-P2WPKH) for two reasons: it is cheaper on-chain (smaller witness data counts at a discount), and bech32 addresses have built-in error detection. BIP-84 is the default path in most modern wallets — BitBox02, Coldcard, Sparrow, and many others use `m/84'` as their primary account path.
