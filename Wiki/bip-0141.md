# BIP-0141: Segregated Witness (Consensus layer)

## Summary

This BIP defines a new structure called a "witness" that is committed to blocks separately from the transaction Merkle tree, moving scripts and signatures into this new structure.

### Zweck

Segregated Witness (SegWit) is the most significant Bitcoin protocol upgrade since its launch. Activated in August 2017, it restructured Bitcoin transactions in several fundamental ways.

Signature and script data (the "witness") is moved to a separate data structure alongside transactions but not included in the txid hash. This eliminates transaction malleability for SegWit inputs — altering the witness data changes the witness txid (wtxid) but not the txid. Dependent transactions referencing a txid are safe.

SegWit replaced the 1MB block size limit with a 4 million weight unit limit. Each byte of witness data costs 1 weight unit; each byte of non-witness data costs 4. This discounts witness data (signatures) by 75%, making SegWit transactions cheaper per byte of economic data. The effective block size for a fully-SegWit block is roughly 1.7–2MB.

SegWit introduced "witness program" outputs with a version byte. Version 0 enables P2WPKH and P2WSH (BIP-141/143); version 1 enables Taproot (BIP-341). Future script upgrades can use higher version numbers as soft forks without changing existing validation rules.

For backward compatibility, SegWit outputs evaluate to "anyone can spend" under old script rules. Old nodes relay and mine SegWit transactions without understanding witness data, making the upgrade a soft fork.
