# BIP-0434: Peer Feature Negotiation

## Summary

This BIP proposes a general peer feature negotiation mechanism for the Bitcoin P2P protocol, allowing nodes to discover each other's supported protocol features before exchanging application messages.

### Zweck

Bitcoin's P2P protocol has grown incrementally: each new feature (BIP-152 compact blocks, BIP-155 addrv2, BIP-339 wtxid relay, BIP-324 v2 transport) added its own handshake messages. There's no unified mechanism for a node to advertise all its capabilities at once or for peers to request specific feature sets.

BIP-434 proposes a general-purpose feature negotiation message that lets peers declare supported features in a compact bitmask or list format during connection setup. This would reduce the number of separate negotiation messages (one per feature) to a single exchange at handshake time.

As a 2026 Draft, this is forward-looking infrastructure for a P2P protocol that continues to grow.

## Open Questions
