# BIP-0373: MuSig2 PSBT Fields

## Summary

This document proposes additional PSBT fields for BIP-327 MuSig2 multi-signature data, including participant keys, public nonces, and partial signatures.

### Zweck

MuSig2 (BIP-327) requires a two-round signing protocol: first, each participant generates and shares a public nonce; second, each participant produces a partial signature using their nonce and the aggregated nonce. In a PSBT workflow with offline signers, this multi-round exchange needs to be carried in the PSBT file format.

BIP-373 adds three sets of new PSBT fields for MuSig2:
- **Participant public keys**: the individual signer pubkeys that were aggregated into the output key
- **Public nonces**: each signer's contribution to the aggregate nonce (Round 1 output)
- **Partial signatures**: each signer's partial signature (Round 2 output, computed after all nonces are known)

The Combiner role (defined in BIP-174) merges PSBTs from all participants after each round. The Finaliser computes the aggregate signature from the partial signatures once all are collected.

Together with BIP-327 (protocol) and BIP-328 (key derivation), BIP-373 completes the PSBT support needed for hardware wallets to participate in MuSig2 multisig workflows.
