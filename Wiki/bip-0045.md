# BIP-0045: Structure for Deterministic P2SH Multisignature Wallets

## Summary

This BIP defines a structure for hierarchical deterministic P2SH multi-party multi-signature wallets based on BIP-0032 and BIP-0043.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Complete (abgeschlossen)  
**Autor(en):** Manuel Araoz, Ryan X. Charles, Matias Alejo Garcia  
**Veröffentlicht:** 2014-04-25  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0045.mediawiki

### Zweck

BIP-0045 defines how multi-party multisig wallets should derive keys using BIP-32, specifically for cases where each cosigner has their own HD seed and contributes a public key. The path structure is `m/45'/cosigner_index/change/address_index`.

The `45'` purpose distinguishes this from BIP-44 single-key accounts. The `cosigner_index` level assigns each participant a fixed index — cosigner 0's keys are at index 0, cosigner 1's at index 1, and so on. All cosigners share the same `change` and `address_index` path, so each generates the same multisig output script by combining keys from all cosigners at the same index.

BIP-45 assumes all cosigners share a common xpub. The more modern successor for multisig coordination is BIP-48 (which handles multiple script types including P2WSH and P2TR) and BIP-129 (Bitcoin Secure Multisig Setup protocol).
