# BIP-0089: Chain Code Delegation

## Summary

Chain Code Delegation (CCD) allows a privileged participant in a multisig to withhold BIP-32 chain codes from non-privileged participants, supplying per-input tweaks at signing time instead.

### Zweck

BIP-0089 addresses a privacy problem in multisig wallet coordination: in standard multisig, all cosigners receive each other's xpubs. This means each cosigner can derive all past and future addresses for the wallet, see all balances, and correlate signing activity across sessions. A cosigner knows everything about the wallet even between signings.

With Chain Code Delegation, one "privileged" participant (e.g. a coordinator) withholds the chain codes. Non-privileged signers receive only per-input scalar tweaks at signing time, which is enough to produce a valid signature but reveals nothing about other inputs or addresses. The signer can verify the tweak is correctly derived (via a zero-knowledge proof or commitment scheme) without needing the chain code.

Relevant for institutional custody and privacy-sensitive multisig setups where cosigners should be limited to "can sign specific transactions" without gaining full wallet visibility. Still in Draft as of 2025.
