# BIP-0324: Version 2 P2P Encrypted Transport Protocol

## Summary

This document proposes a new Bitcoin P2P transport protocol with opportunistic encryption, mild bandwidth reduction, and upgrade negotiation.

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Dhruv Mehta, Tim Ruffing, Jonas Schnelli, Pieter Wuille  
**Veröffentlicht:** 2019-03-08  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0324.mediawiki

### Zweck

BIP-0324 replaces Bitcoin's plaintext P2P protocol with an encrypted transport, implemented in Bitcoin Core v26 (2023). The protocol uses ECDH with x-only public keys (consistent with BIP-340/Taproot) to establish a shared secret, then encrypts all traffic with ChaCha20-Poly1305.

The encryption is opportunistic — it protects against passive surveillance (ISPs, surveillance nodes recording traffic) but not active man-in-the-middle attacks without an additional authentication step. A session ID is derived from the ECDH handshake and can be used for out-of-band verification (e.g., comparing session IDs over a separate channel) to detect active MITM.

Beyond privacy, BIP-324 provides a mild bandwidth saving (~20% in tests) because the new protocol uses a more compact message framing. It also allows feature negotiation in the handshake, making future protocol extensions easier to add.
