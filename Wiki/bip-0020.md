# BIP-0020: URI Scheme

## Summary

This BIP proposes a URI scheme for making Bitcoin payments.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Autor(en):** Luke Dashjr  
**Veröffentlicht:** 2011-01-10  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0020.mediawiki

### Zweck

BIP-0020 was the first attempt to standardise `bitcoin:` URIs for payment requests. The format allowed encoding a destination address plus optional parameters (amount, label, message) in a single clickable link or scannable QR code.

It was closed because the specification had inconsistencies and ambiguities — particularly around how amounts should be encoded and how parameters should be parsed. BIP-21 replaced it with a cleaner, more precisely defined specification. The core concept (a `bitcoin:` URI scheme) carried forward unchanged; BIP-20 and BIP-21 differ only in the technical details of the format.
