# BIP-0115: Generic anti-replay protection using Script

## Summary

This BIP describes a new opcode (OP_CHECKBLOCKATHEIGHT) for the Bitcoin scripting system that allows construction of transactions which are valid only on specific blockchains.

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Autor(en):** Luke Dashjr  
**Veröffentlicht:** 2016-09-23  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0115.mediawiki

### Zweck

BIP-0115 proposed `OP_CHECKBLOCKATHEIGHT` — an opcode that checks that the block hash at a specified height matches an expected value. This would allow transactions to be bound to a specific chain: a transaction including this opcode with the genesis block hash of Bitcoin mainnet would be invalid on any chain that forked before that block (or on testnet).

The motivation was replay attack protection for chain splits. If Bitcoin were to fork (as it did in 2017 with Bitcoin Cash), transactions signed on one chain could be replayed on the other. `OP_CHECKBLOCKATHEIGHT` would let users cryptographically bind their transactions to a specific chain's history.

The proposal was closed; replay protection in the BCH split was handled through other mechanisms (different `SIGHASH` flag, different address format). The opcode itself was considered too niche and complex for general activation.
