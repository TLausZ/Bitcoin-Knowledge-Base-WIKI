# BIP-0049: Derivation scheme for P2WPKH-nested-in-P2SH based accounts

## Summary

This BIP defines the derivation scheme for HD wallets using the P2WPKH-nested-in-P2SH (BIP 141) serialization format for segregated witness transactions.

### Zweck

BIP-0049 defines the derivation path for "wrapped SegWit" addresses — the P2SH-wrapped P2WPKH format that was the transitional SegWit address type, producing addresses starting with `3`. The path is `m/49'/coin_type'/account'/change/index`.

Wrapped SegWit (P2SH-P2WPKH) was designed as a bridge: it gets the transaction weight discount from SegWit (inputs are cheaper) while using the `3...` address format that older wallets could already send to. The `49'` purpose field tells a restoring wallet to scan for this specific address type rather than legacy (`44'`) or native SegWit (`84'`).

In practice, wrapped SegWit was a stepping stone. Native bech32 addresses (BIP-84) are now preferred because they are cheaper (smaller on-chain footprint), have built-in error detection, and don't require the P2SH wrapping overhead. Many modern wallets have moved from `m/49'` to `m/84'` as their default.
