# BIP-0140: Normalized TXID

## Summary

This BIP describes the use of normalized transaction IDs (NTXIDs) to eliminate transaction malleability by computing the txid over the transaction data with signatures stripped.

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Themen:** bips

**Veröffentlicht:** 2015-10-14

### Zweck

BIP-0140 proposed fixing transaction malleability by computing the txid over the transaction data *without* the signature scripts (scriptSig). Since malleability attacks work by modifying signature encoding to produce a different txid while keeping the signatures valid, a txid computed over signature-free data would be immutable.

The approach was conceptually clean but required adding a new "normalized txid" field alongside the existing txid in transaction references, which complicated the consensus and serialisation rules. SegWit (BIP-141) achieved the same outcome more elegantly: by moving witness data (signatures) out of the transaction body that is hashed for the txid, SegWit outputs are inherently non-malleable. BIP-140 was closed when SegWit made it redundant.
