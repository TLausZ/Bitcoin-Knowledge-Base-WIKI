# BIP-0382: Segwit Output Script Descriptors

## Summary

This document specifies `wpkh()` and `wsh()` output script descriptors for native SegWit outputs.

## Body

**Typ:** Informational  
**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)  
**Themen:** bips, self-custody, privacy
**Autor(en):** Pieter Wuille, Ava Chow  
**Veröffentlicht:** 2021-06-27  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0382.mediawiki

### Zweck

BIP-382 defines the native SegWit descriptor functions:

- `wpkh(KEY)` — Pay-to-Witness-PubKey-Hash (P2WPKH): native SegWit single-sig, producing `bc1q...` addresses. The standard single-sig format for most modern wallets.
- `wsh(SCRIPT)` — Pay-to-Witness-Script-Hash (P2WSH): native SegWit script hash. Used for SegWit multisig and complex scripts, also producing `bc1q...` addresses.

`wsh()` is commonly combined with `multi()` (BIP-383) to describe native SegWit multisig: `wsh(multi(2, KEY1, KEY2, KEY3))`. These can also be nested inside `sh()` (from BIP-381) to produce the P2SH-wrapped variants for backward compatibility.
