# BIP-0048: Multi-Script Hierarchy for Multi-Sig Wallets

## Summary

This BIP defines a logical hierarchy for deterministic multi-sig wallets based on BIP-0067, BIP-0032, BIP-0043, and BIP-0044.

### Zweck

BIP-0048 is the modern standard for multisig HD derivation, extending BIP-45 to support multiple script types. The path is `m/48'/coin'/account'/script_type'/change/index`, where `script_type'` distinguishes between P2SH-P2WSH (`1'`), native P2WSH (`2'`), and P2TR (`3'` for Taproot multisig).

Over BIP-45, the `script_type'` level — it means a single seed can hold segregated accounts for legacy-wrapped SegWit multisig, native SegWit multisig, and Taproot multisig without path collisions. Hardware wallet vendors (Ledger, Trezor, Coldcard, Bitbox02) have standardised on BIP-48 paths for multisig coordination.

BIP-67 deterministic key sorting is expected to be used with BIP-48 to ensure all cosigners produce the same output script ordering regardless of the order in which they enter their xpubs.
