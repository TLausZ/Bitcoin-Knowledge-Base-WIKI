# BIP-0123: BIP Classification

## Summary

This document describes a classification scheme for BIPs by system layers, with lower numbered layers involving more intricate interoperability requirements.

## Body

**Typ:** Process  
**Layer:** unspecified  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Eric Lombrozo  
**Veröffentlicht:** 2015-08-26  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0123.mediawiki

### Zweck

BIP-0123 introduced the formal layer classification system used in the BIP metadata — the "Layer" field seen in every BIP header. It defines five layers in order of protocol depth:

1. **Consensus** — changes requiring all full nodes to upgrade (soft forks, hard forks)
2. **Peer Services** — P2P network protocol extensions
3. **API/RPC** — interfaces for applications to interact with nodes
4. **Applications** — wallet formats, address schemes, higher-level protocols

The classification helps readers quickly understand a BIP's scope and what software needs updating for adoption. A Consensus-layer BIP affects every node; an Applications-layer BIP might only affect wallet software. BIP-123 is referenced by BIP-2 and BIP-3 as part of the standard BIP template.
