# BIP-0330: Transaction announcements reconciliation

## Summary

This document specifies a P2P protocol extension for reconciliation of transaction announcements between two nodes, a building block for Erlay — efficient transaction relay using set reconciliation.

### Zweck

Bitcoin nodes currently flood the network with transaction announcements: when a node sees a transaction, it sends `inv` messages to all peers. For a network with thousands of nodes, each transaction generates enormous redundant traffic — each node announces the same transaction to all of its peers, most of whom already have it.

BIP-330 proposes set reconciliation as an alternative. Two peers maintain a "reconciliation set" of transactions they intend to announce. Periodically, they exchange compact sketches of their sets using IBLT (Invertible Bloom Lookup Tables), allowing each to identify which transactions the other has that they don't. They request only the missing ones.

This is the "Erlay" protocol: in simulations, it reduces transaction relay bandwidth by ~40% while maintaining similar propagation speed. The approach allows nodes to maintain more peer connections (increasing network connectivity and Eclipse attack resistance) with the same bandwidth budget.
