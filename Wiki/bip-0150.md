# BIP-0150: Peer Authentication

## Summary

This BIP describes a way for peers to authenticate to other peers to guarantee node ownership and allow access to additional node services, without the possibility of fingerprinting.

## Body

### Zweck

BIP-0150 proposed a challenge-response authentication scheme for Bitcoin P2P connections. A node could establish a "trusted peer" relationship with another node using a static identity key, allowing selective access to services (e.g., a private node that only serves authenticated peers). The authentication was built on top of BIP-151's encrypted transport.

The design was careful about fingerprinting: failed authentication attempts look identical to a non-responding peer, so an attacker probing the network can't distinguish authenticated-only nodes from ordinary nodes.

Both BIP-150 and BIP-151 were closed because BIP-324 (v2 P2P transport) took a different approach to the underlying problems — encryption and authentication are handled at the transport layer via Elliptic Curve Diffie-Hellman, without requiring a separate authentication protocol.
