# BIP-0021: URI Scheme

## Summary

This BIP proposes a URI scheme for making Bitcoin payments.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Themen:** bips
**Autor(en):** Nils Schneider, Matt Corallo  
**Veröffentlicht:** 2012-01-29  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0021.mediawiki

### Zweck

BIP-0021 is the active Bitcoin URI scheme standard, replacing BIP-20. The format is:

```
bitcoin:<address>[?amount=<value>][&label=<label>][&message=<msg>]
```

Amount is expressed in BTC as a decimal. Extra parameters are allowed; unknown parameters must be ignored (for forward compatibility) unless prefixed with `req-`, in which case a wallet that doesn't understand the parameter must refuse the payment.

This scheme is how QR codes in wallets work. Scanning a QR code at a merchant launches your wallet with the address and amount pre-filled. Extensions built on BIP-21 include BIP-72 (payment protocol URL), BIP-321 (URI extensions for new address types), and BIP-353 (DNS-resolved payment instructions). Despite its "Closed" status in the BIP repo (meaning the document is finalized, not revoked), BIP-21 remains the de facto standard.
