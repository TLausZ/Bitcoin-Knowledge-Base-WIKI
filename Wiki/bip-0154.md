# BIP-0154: Rate Limiting via peer specified challenges

## Summary

An anti-DoS system which provides additional service for peers which perform proof of work.

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Themen:** bips
**Autor(en):** Karl-Johan Alm  
**Veröffentlicht:** 2017-04-12  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0154.mediawiki

### Zweck

BIP-0154 proposed a mechanism for nodes to rate-limit resource-hungry requests by requiring requesting peers to complete small proof-of-work challenges first. A node could issue a challenge (a hash puzzle) before serving data-intensive requests; peers that solve the puzzle get priority service.

The motivation was protecting nodes from DoS attacks that flood them with expensive requests (e.g., getdata for historical blocks). By requiring a small computational cost, the attacker's cost scales with the attack volume while honest peers with mostly idle CPUs experience little inconvenience.

The proposal was closed — the BIP-37 Bloom filter DoS concern (which this partially addressed) was handled instead by disabling or rate-limiting BIP-37 in Bitcoin Core. The broader PoW-challenge approach didn't gain traction.

## Open Questions
