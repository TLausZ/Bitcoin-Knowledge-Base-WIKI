# BIP-0042: A finite monetary supply for Bitcoin

## Summary

Although it is widely believed that Satoshi was an inflation-hating goldbug he never said this, and in fact programmed Bitcoin's money supply to grow indefinitely, forever. This BIP fixes the supply to be truly finite.

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)  
**Status:** Deployed (aktiviert und im Einsatz)

### Zweck

BIP-0042 is an April Fools' joke that nonetheless fixed a real bug. The humorous framing describes Bitcoin's supply as "growing indefinitely" due to how the original C++ code handled the halving schedule: the block subsidy is computed by right-shifting 50 BTC by the number of halvings. In C++, right-shifting a 64-bit integer by 64 or more positions is undefined behaviour — different compilers produce different results. On most compilers at the time, this would eventually produce non-zero (and thus inflationary) results after 64 halvings, roughly 300 years from now.

The actual fix (buried in the joke): add an explicit check that the subsidy returns zero once the shift count exceeds 63. Bitcoin Core had already implemented this check, so BIP-42 is really a post-hoc documentation of behaviour that was already correct. The "joke BIP with a real patch" format was Pieter Wuille's way of drawing attention to an obscure edge case without causing unnecessary alarm.
