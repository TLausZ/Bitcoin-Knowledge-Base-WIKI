# BIP-0442: OP_PAIRCOMMIT

## Summary

This BIP proposes OP_PAIRCOMMIT, an opcode that computes a hash commitment over a pair of stack elements and pushes the result.

### Zweck

OP_PAIRCOMMIT pops two elements from the stack and pushes `SHA256(element1 || element2)`, a hash commitment over the pair. It's a building block for the "LNhance" proposal — a bundle of opcodes (OP_CTV/BIP-119, OP_CHECKSIGFROMSTACK/BIP-348, OP_PAIRCOMMIT) designed to enable Lightning Network symmetry channels (LN-Symmetry, formerly eltoo).

The specific use case in LN-Symmetry: each channel state update involves a "settlement transaction" that spends a "update transaction." With a commitment scheme like OP_PAIRCOMMIT, the update transaction can commit to both the state number and the settlement transaction ID without requiring additional script complexity. This enables the "rebinding" of settlement transactions to whichever update transaction settles on-chain.

OP_PAIRCOMMIT is simpler than OP_CAT for this specific use — it hashes exactly two elements without enabling arbitrary-length concatenation, which limits its scope and risk surface.
