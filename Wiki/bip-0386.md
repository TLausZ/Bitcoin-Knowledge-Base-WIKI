# BIP-0386: tr() Output Script Descriptors

## Summary

This document specifies `tr()` output script descriptors for P2TR (Taproot) outputs.

## Body

**Typ:** Informational  
**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)  
**Themen:** bips, self-custody, privacy
**Autor(en):** Pieter Wuille, Ava Chow  
**Veröffentlicht:** 2021-06-27  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0386.mediawiki

### Zweck

`tr(KEY)` is the Taproot descriptor: it takes an internal key and optionally a script tree, producing a P2TR output. The internal key handles key-path spending; the optional tree adds script-path spending conditions.

Examples:
- `tr(xpub.../0/*)` — simple Taproot single-sig, key-path only. Produces standard `bc1p...` addresses.
- `tr(KEY, pk(KEY2))` — Taproot with a single script-path leaf: key-path is primary, script-path with `KEY2` as fallback.
- `tr(KEY, {pk(KEY2), {pk(KEY3), pk(KEY4)}})` — Taproot with a two-level script tree (Merkle tree with 3 leaves).

The tree syntax `{A, B}` describes a binary tree node. This directly represents BIP-341's Merkle tree structure. The `tr()` descriptor is the only way to fully represent a Taproot spending policy in the descriptor language, including both key-path and all script-path alternatives.
