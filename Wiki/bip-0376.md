# BIP-0376: Spending Silent Payment outputs with PSBTs

## Summary

This document proposes additional PSBTv2 per-input fields for Silent Payment (BIP-352) tweak data required to spend a Silent Payment output.

### Zweck

A Silent Payment output is a P2TR output whose key is derived as: output_key = spend_key + tweak·G, where tweak = hash(shared_secret || output_index). To spend it, the recipient needs the spend_key private key *plus* the tweak value — which depends on the original sender's inputs and thus isn't stored in the blockchain.

When spending a Silent Payment output via PSBT, the signer needs to know the tweak to reconstruct the private key for the output. BIP-376 adds a new per-input PSBT field to carry this tweak, alongside the DLEQ proof that the tweak was computed correctly.

This completes the Silent Payments PSBT support: BIP-375 covers sending to SP addresses; BIP-376 covers spending from them.
