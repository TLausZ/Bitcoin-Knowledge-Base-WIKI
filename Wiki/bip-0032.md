# BIP-0032: Hierarchical Deterministic Wallets

## Summary

This document describes hierarchical deterministic wallets (or "HD Wallets"): wallets which can be shared partially or entirely with different systems, each with or without the ability to spend coins.

### Zweck

BIP-0032 is the foundation of all modern Bitcoin wallets. It defines how to derive an entire tree of key pairs from a single 512-bit seed, using a chain of HMAC-SHA512 derivations. Each node in the tree is identified by a path like `m/0/1/2` and produces a unique private/public key pair.

Knowing a parent's extended public key (xpub) lets you derive all child public keys without ever seeing the private keys. This makes it possible to give a watch-only key to an observer (e.g. an accounting system or a hardware wallet companion app) who can generate new receiving addresses without being able to spend funds.

Hardened derivation (marked with a `'`) breaks this relationship: a hardened child's private key cannot be derived from the parent xpub alone, providing compartmentalisation between accounts. BIP-32 defines both normal and hardened derivation.

Every HD wallet in use today (hardware wallets, mobile wallets, multisig coordinators) builds on BIP-32. BIP-39 defines how to generate the seed from a mnemonic; BIP-43/44/49/84/86 define standard path structures on top of BIP-32.
