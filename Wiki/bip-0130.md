# BIP-0130: sendheaders message

## Summary

Add a new message, "sendheaders", which indicates that a node prefers to receive new block announcements via a "headers" message rather than an "inv".

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Deployed (aktiviert und im Einsatz)  
**Themen:** bips
**Autor(en):** Suhas Daftuar  
**Veröffentlicht:** 2015-05-08  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0130.mediawiki

### Zweck

Before BIP-130, when a node learned of a new block, it would send an `inv` message to its peers containing just the block hash. Peers that wanted the block would then send `getdata`, and only then receive the headers and transactions. This was two round trips before a peer could start validating the block.

BIP-130 adds a `sendheaders` message: a node can indicate it prefers to receive block *headers* directly (skipping the `inv`/`getdata` round trip). When a peer mines or relays a new block, it sends the 80-byte header immediately to `sendheaders` peers. Peers can start validating the header and request the full block immediately, saving one round trip. This reduces block propagation latency — important for miners and well-connected relay nodes where every millisecond of propagation delay has economic consequences.
