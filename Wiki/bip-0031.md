# BIP-0031: Pong message

## Summary

This document describes a trivial protocol extension that makes it easier for clients to detect dead peer connections.

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Deployed (aktiviert und im Einsatz)  
**Themen:** bips
**Autor(en):** Mike Hearn  
**Veröffentlicht:** 2012-04-11  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0031.mediawiki

### Zweck

BIP-0031 added a `pong` message to the Bitcoin P2P protocol in response to an existing `ping` message. Before this BIP, Bitcoin had a `ping` message but no corresponding `pong` — there was no way to confirm a peer had received and processed the ping. Nodes could only detect dead connections by waiting for them to time out.

The BIP added a nonce field to `ping` and a `pong` response that echoes the nonce back. This lets a node confirm that a specific peer is alive and responsive. The round-trip time can also be used to measure latency to peers. A simple extension, but it underpins basic connection health monitoring in every Bitcoin node.
