# BIP-0111: NODE_BLOOM service bit

## Summary

This BIP extends BIP-37 (bloom filtering) by defining a service bit to allow peers to advertise explicit bloom filter support, and bumps the protocol version.

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Matt Corallo, Peter Todd  
**Veröffentlicht:** 2015-08-20  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0111.mediawiki

### Zweck

BIP-0111 added a `NODE_BLOOM` service bit (bit 2 in the service flags field) to let full nodes explicitly advertise whether they support BIP-37 bloom filter requests. Before BIP-111, all nodes implicitly supported bloom filtering if their protocol version was high enough — there was no opt-out.

This mattered because BIP-37 had been found to have DoS vulnerabilities: a malicious SPV client could send crafted bloom filters that forced nodes to do expensive work. BIP-111 allowed node operators to disable bloom filter serving (`-peerbloomfilters=0` in Bitcoin Core) while still announcing their other capabilities. Nodes without `NODE_BLOOM` can legitimately reject bloom filter requests from peers.

With BIP-157/158 (compact block filters) now the preferred light client mechanism, the `NODE_BLOOM` bit is increasingly an opt-in for legacy SPV support.
