# BIP-0177: Redefine Bitcoin's Base Unit

## Summary

This BIP proposes redefining "bitcoin" so that the base unit (currently called a satoshi) becomes the primary reference unit — eliminating the convention of eight decimal places.

### Zweck

At the protocol level, Bitcoin has no decimals — the ledger counts integer units. "Bitcoin" with eight decimal places is a presentation layer convention inherited from Satoshi, where 1 BTC = 100,000,000 base units. BIP-177 proposes inverting this: define 1 bitcoin = 1 base unit (what is currently called a satoshi). The 21 million BTC supply cap becomes 2.1 quadrillion bitcoins.

The practical argument: as BTC price increases, amounts like 0.00001 BTC become unwieldy. If 1 bitcoin = 1 satoshi, amounts are always whole numbers, consistent with how the protocol actually works, and intuitively align with "you have X bitcoins" rather than fractions.

The proposal is contentious. Critics argue it creates confusion with the current 1 BTC = 100M sat definition, would require all wallets and exchanges to update their display logic simultaneously, and that the problem is already solved by satoshis as a denomination. As a 2025 Draft, it remains open.
