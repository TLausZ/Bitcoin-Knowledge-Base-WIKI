# BIP-0106: Dynamically Controlled Bitcoin Block Size Max Cap

## Summary

This BIP proposes replacing the fixed one megabyte maximum block size with a dynamically controlled maximum that may increase or decrease with difficulty change depending on various network factors.

### Zweck

BIP-0106 was another algorithmic block size proposal from the 2015 block size debate. It offered two variants: one where the block size limit adjusts based solely on how full recent blocks were; another where transaction fee revenue collected by miners also influences the limit. The goal was a self-regulating limit that would expand when demand was high and contract when demand was low.

Like BIPs 100, 104, and 105, it was closed because all hard fork block size proposals failed to achieve consensus, and SegWit provided a different path to increased capacity.
