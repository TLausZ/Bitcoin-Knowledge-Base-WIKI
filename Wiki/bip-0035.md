# BIP-0035: mempool message

## Summary

Make a network node's transaction memory pool accessible via a new "mempool" message. Extend the existing "getdata" message behavior to permit accessing the transaction memory pool.

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Deployed (aktiviert und im Einsatz)  
**Themen:** bips
**Autor(en):** Jeff Garzik  
**Veröffentlicht:** 2012-08-16  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0035.mediawiki

### Zweck

BIP-0035 added the `mempool` P2P message, which allows a node to request the list of txids currently in a peer's mempool. The peer responds with an `inv` message listing those txids; the requester can then fetch specific transactions it doesn't have using `getdata`.

This is useful for newly connecting nodes: when a node joins the network, it has no pending transactions. Sending `mempool` to peers lets it quickly populate its mempool from existing network participants rather than waiting for new transactions to propagate. It is also used by SPV clients (BIP-37) to retrieve unconfirmed transactions relevant to their addresses. The message was later complemented by BIP-133 (fee filter) to allow nodes to only share transactions meeting a fee threshold.
