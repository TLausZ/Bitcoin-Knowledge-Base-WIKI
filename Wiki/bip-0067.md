# BIP-0067: Deterministic Pay-to-script-hash multi-signature addresses through public key sorting

## Summary

This BIP describes a method to deterministically generate multi-signature pay-to-script-hash transaction scripts through lexicographic public key sorting.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Complete (abgeschlossen)  
**Autor(en):** Thomas Kerin, Jean-Pierre Rupp, Ruben de Vries  
**Veröffentlicht:** 2015-02-08

### Zweck

Without a deterministic key ordering rule, two cosigners providing the same public keys could produce different multisig scripts depending on the order in which they entered the keys. A 2-of-2 multisig with keys [A, B] produces a different P2SH address than [B, A], even though both would require the same two signatures to spend.

BIP-0067 specifies that public keys in multisig scripts must be sorted lexicographically by their compressed serialisation before building the script. This means any wallet, given the same set of public keys, will deterministically produce the same address — regardless of the order in which users entered them. This is required for cross-wallet multisig interoperability.

BIP-67 sorting is expected to be used alongside BIP-44/48 for multisig HD wallets, and it underpins multisig coordinator tools like Specter, Sparrow, and hardware wallet coordination flows.
