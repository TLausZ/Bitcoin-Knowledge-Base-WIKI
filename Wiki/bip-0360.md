# BIP-0360: Pay-to-Merkle-Root (P2MR)

## Summary

This document proposes a new output type — Pay-to-Merkle-Root (P2MR) — resistant to quantum attacks from CRQCs (Cryptographically Relevant Quantum Computers). P2MR operates like P2TR but with the key-path spend removed, relying only on script-path spends secured by hash-based commitments.

### Zweck

Taproot (P2TR) outputs have a key-path spend that requires revealing an elliptic curve public key and signature. If a sufficiently powerful quantum computer ever exists, it could derive private keys from public keys — breaking ECDSA and Schnorr alike. Funds in P2TR outputs are vulnerable because the internal key is embedded in the output.

BIP-360 proposes P2MR: a Taproot-like output that commits only to a Merkle tree root (no internal key), eliminating the key-path spend entirely. Spending requires revealing a leaf script from the Merkle tree and satisfying it. The leaf scripts themselves can use hash-based commitments or future quantum-resistant signature schemes.

This provides a migration path: users can move funds from ECDSA/Schnorr outputs into P2MR outputs now, before quantum computers are a practical threat. The scripts within P2MR can be updated via soft fork as quantum-resistant signature schemes mature.
