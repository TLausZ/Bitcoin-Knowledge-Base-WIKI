# BIP-0014: Protocol Version and User Agent

## Summary

This BIP standardises how Bitcoin nodes identify their software version and protocol support in the P2P handshake.

## Body

**Typ:** Specification  
**Layer:** Peer Services  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Amir Taaki, Patrick Strateman  
**Veröffentlicht:** 2011-11-10  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0014.mediawiki

### Zweck

BIP-0014 standardised the format of user-agent strings and protocol version numbers exchanged during the P2P handshake (`version` message). Before this, different clients announced version information in incompatible formats, making it impossible to reliably identify what software a peer was running.

The standard format for the user-agent string is `/Name:Version/` (e.g. `/Satoshi:0.21.0/` for Bitcoin Core). Sub-components can be added with additional slashes, e.g. `/BitcoinJ:0.13/Android:6.0/`. The protocol version number is a separate integer that indicates which P2P features the node supports — this is distinct from the software version.

In practice, user-agent strings help network operators understand what software the network is running, allow researchers to study client diversity, and help identify nodes running outdated software.
