# BIP-0151: Peer-to-Peer Communication Encryption

## Summary

This BIP describes an alternative way for peers to encrypt their communication between a selective subset of remote peers.

### Zweck

Bitcoin's P2P protocol transmits all data in plaintext, including transactions and block data. This allows network observers (ISPs, state actors, surveillance nodes) to monitor which transactions a node broadcasts, correlating them with IP addresses to deanonymise users.

BIP-0151 proposed adding encryption to P2P connections using ECDH key exchange and ChaCha20-Poly1305. The encryption was opportunistic — it protects against passive monitoring but not active man-in-the-middle attacks unless combined with BIP-150 peer authentication.

BIP-151 was eventually superseded by BIP-324 (v2 transport), which improved on BIP-151's design: better cipher suite choices, ECDH using x-only public keys (consistent with Taproot), and explicit session ID support for out-of-band authentication.
