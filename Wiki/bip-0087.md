# BIP-0087: Hierarchy for Deterministic Multisig Wallets

## Summary

This BIP defines a hierarchy for deterministic multisig wallets based on BIP-0032 and BIP-0043.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Complete (abgeschlossen)

**Veröffentlicht:** 2020-03-11  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0087.mediawiki

### Zweck

BIP-0087 defines a derivation path for deterministic multisig wallets: `m/87'/coin_type'/account'`. Unlike BIP-45 (which embedded cosigner index in the path) or BIP-48 (which adds a script type level), BIP-87 takes a simpler approach: each cosigner contributes an xpub at this path, and the coordinator combines them with a descriptor to define the full wallet policy.

In BIP-87, the descriptor (the wallet policy, including the M-of-N threshold and all cosigner xpubs) is the complete specification of the wallet. The derivation path just provides the xpub; everything else lives in the descriptor. This is closely aligned with the descriptor wallet model adopted by Bitcoin Core.

In practice, BIP-48 has become more widely adopted among hardware wallets, but BIP-87 represents the cleaner conceptual model where wallet policy and key derivation are clearly separated.
