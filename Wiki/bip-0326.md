# BIP-0326: Anti-fee-sniping in taproot transactions

## Summary

This document proposes wallet behaviour using BIP-341 Taproot to increase the anonymity set for off-chain protocols using PTLCs (CoinSwap, Lightning, Discrete Log Contracts).

### Zweck

Fee sniping is a miner attack where a miner, upon finding a block, mines an alternate block at the same height (with the same transactions plus additional high-fee transactions it sees), trying to replace the honest block and collect more fees. BIP-68 and BIP-125 allow wallets to set nLockTime to the current block height to prevent their transactions from being included in older reorgs — this anti-fee-sniping technique is already used by many wallets.

BIP-326 proposes that Taproot wallets using PTLCs (Point Time-Locked Contracts) adopt this same nLockTime or nSequence behaviour to deter fee sniping. The secondary benefit: if on-chain Taproot transactions and off-chain PTLC settlement transactions use the same nLockTime/nSequence patterns, it becomes harder for chain analysis to distinguish regular wallet transactions from Lightning or CoinSwap settlements. The anonymity set grows because all Taproot wallets look the same.
