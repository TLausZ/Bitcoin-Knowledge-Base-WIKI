# BIP-0136: Bech32 Encoded Tx Position References

## Summary

This document proposes a convenient, human-usable encoding to refer to a confirmed transaction position within the Bitcoin blockchain — known as "TxRef."

### Zweck

BIP-0136 defines TxRef — a compact, human-readable way to reference a transaction's position in the Bitcoin blockchain using block height and transaction index. Example: `tx1:rqqq-qqqq-qmhu-qhp` (mainnet genesis coinbase). The encoding uses bech32 (from BIP-173) and includes a network identifier prefix.

Unlike a txid, which is a hash of the transaction data, a TxRef encodes *where* in the blockchain the transaction is — not what it is. This means TxRefs are shorter (~20 characters vs 64-character txids) and readable in human conversation. The tradeoff: TxRefs are only valid for *confirmed* transactions, and they can't be computed until the transaction is in a block.

TxRef was used in the DID (Decentralised Identifier) specification for Bitcoin-based identities, and has seen some use in Decentralised Identity contexts. As a Draft, it hasn't achieved broad adoption.
