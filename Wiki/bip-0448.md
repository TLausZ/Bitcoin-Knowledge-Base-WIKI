# BIP-0448: Taproot-native (Re)bindable Transactions

## Summary

This BIP proposes a new signature hash type enabling "rebindable" transactions — pre-signed transactions that can be bound to different UTXOs at spend time.

### Zweck

Bitcoin signatures normally commit to the specific UTXO being spent (its txid and output index). This is by design — it prevents signatures from being reused across different UTXOs. But for certain protocols, the ability to rebind a pre-signed transaction to a different-but-equivalent UTXO is desirable.

The target use case is Lightning Network Symmetry (eltoo). In LN-Symmetry, when a channel state is updated, the new update transaction must be able to spend *any* earlier update transaction, not just a specific one. This requires a signature that doesn't commit to the input UTXO's txid. BIP-118's SIGHASH_ANYPREVOUT addresses this for the existing sighash framework; BIP-448 proposes a Taproot-native equivalent with cleaner semantics.

"Rebindable" in BIP-448 means the signature covers the output script, value, and structure — but not the specific txid being spent. It can only bind to UTXOs with matching script and value, preventing misuse.
