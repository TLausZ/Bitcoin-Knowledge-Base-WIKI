# BIP-0143: Transaction Signature Verification for Version 0 Witness Program

## Summary

This proposal defines a new transaction digest algorithm for signature verification in version 0 witness programs, fixing quadratic hashing and adding input value commitment.

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Johnson Lau, Pieter Wuille  
**Veröffentlicht:** 2016-01-03

### Zweck

BIP-0143 defines the sighash algorithm for SegWit v0 (P2WPKH and P2WSH) inputs, fixing two problems with the legacy algorithm.

In legacy transactions, the signature hash for each input is computed by serialising the entire transaction with that input's script filled in — all other inputs replaced with empty scripts. A transaction with N inputs therefore requires O(N²) hashing. For a 1000-input transaction this means roughly 500,000 hash operations. Hardware wallets verifying such a transaction could require minutes of computation. BIP-143 uses a streaming hash that hashes each input's data once, making sighash computation linear in the number of inputs.

Legacy signatures also don't commit to the value (amount) of the UTXO being spent. A hardware wallet signing a transaction had no way to verify input amounts without downloading and checking each prior transaction — network access an airgapped device cannot have. BIP-143 includes the input value in the signature hash, so the signing device can verify it's not being tricked about how much it's spending.
