# BIP-0098: Fast Merkle Trees

## Summary

This BIP proposes an improved Merkle hash-tree for Bitcoin that addresses the duplicate-entry vulnerability (CVE-2012-2459) and improves performance by avoiding unnecessary double-hashing.

### Zweck

Bitcoin's Merkle tree (used to commit all transactions into a block header) has two known flaws. First, the duplicate-entry vulnerability: when a tree has an odd number of leaves, the last leaf is duplicated to make an even number. This creates an ambiguity where an internal node can be confused with a leaf transaction hash, potentially allowing crafted proofs to verify incorrectly. Second, the tree uses double-SHA256 (SHA256d) throughout, which is slower than necessary.

BIP-98 proposes a new Merkle tree construction using single SHA256 with domain separation tags to distinguish leaf nodes from internal nodes, eliminating the duplicate-entry ambiguity without the performance overhead of double hashing. The proposal was developed in the context of future soft forks (particularly Taproot and script extensions) that would rely on Merkle proofs.

The BIP remains in Draft. Taproot (BIP-341) uses its own Merkle tree construction (Tapscript's script tree uses tagged SHA256 hashing) that achieves similar goals, making BIP-98 as a standalone upgrade less urgent.
