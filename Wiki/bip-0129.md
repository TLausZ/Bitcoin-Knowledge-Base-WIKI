# BIP-0129: Bitcoin Secure Multisig Setup (BSMS)

## Summary

This document proposes a mechanism to set up multisig wallets securely.

### Zweck

BIP-0129 (BSMS — Bitcoin Secure Multisig Setup) solves a critical security problem in multisig wallet setup: if the coordinator's device is compromised during setup, it could substitute one cosigner's xpub with an attacker's xpub. The user would receive a wallet that appears valid but requires the attacker's key to spend.

BSMS provides a protocol for cosigners to securely exchange xpubs and verify the resulting wallet descriptor. Each device gets a "token" (derived from a temporary secret) that is used to encrypt and authenticate the xpub exchange. After all cosigners have provided their xpubs, each device verifies the final descriptor and displays the first few wallet addresses. The cosigners visually compare addresses across all devices — if any device shows different addresses, the setup was tampered with.

This verification step is the key security property: an attacker who substituted a key would produce different addresses on the compromised device, detectable by cross-checking. BSMS is supported by Coldcard, SeedSigner, Sparrow, and other wallets.
