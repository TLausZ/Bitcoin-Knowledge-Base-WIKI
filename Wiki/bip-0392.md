# BIP-0392: Silent Payment Output Script Descriptors

## Summary

This document specifies `sp()` output script descriptors for Silent Payments (BIP-352).

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Draft (noch im Entwurfsstadium)  
**Autor(en):** Craig Raw  
**Veröffentlicht:** 2026-02-06  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0392.mediawiki

### Zweck

Silent Payment addresses (BIP-352) produce P2TR outputs whose key is computed per-payment via ECDH. Standard descriptors (BIP-380+) assume a fixed derivation tree and can't represent outputs whose keys depend on sender input data.

BIP-392 defines `sp(SCAN_KEY, SPEND_KEY)` as a new descriptor type for Silent Payment wallets. Unlike standard descriptors, `sp()` doesn't produce a fixed set of addresses — it describes how to scan the blockchain (using the scan key) and how to spend discovered payments (using the spend key). The wallet must scan each block's transactions to detect incoming payments.

This integrates Silent Payments into the descriptor ecosystem, allowing wallet software to backup and restore Silent Payment wallets in the standard format. As a 2026 Draft, it's part of the growing Silent Payments infrastructure.
