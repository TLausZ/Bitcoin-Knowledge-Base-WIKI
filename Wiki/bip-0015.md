# BIP-0015: Aliases

## Summary

Using vanilla bitcoin, to send funds to a destination, an address is needed. The problem with using addresses is that they are not easy to remember. An analogy can be thought if one were required to enter the IP address of their favourite websites if domain names did not exist.

### Zweck

BIP-0015 was an early proposal to map human-readable names (e.g. `satoshi@bitcoin.org`) to Bitcoin addresses, similar to how DNS maps domain names to IP addresses. The idea was to allow people to send bitcoin to memorable identifiers rather than raw addresses.

The proposal relied on a centralised DNS-like resolution infrastructure, which introduced trust assumptions: the alias resolver could serve a wrong address and redirect payments. This was the core objection that killed the proposal. Additionally, it didn't handle address rotation (re-using the same address leaks privacy), and it didn't specify what entity would run the alias registry.

The same problem space has been revisited many times since: BIP-47 (reusable payment codes), BIP-353 (DNS-based payment instructions using DNSSEC), and Lightning's LNURL-pay all attempt address-like usability without centralised resolution.
