# BIP-0389: Multipath Descriptor Key Expressions

## Summary

This document specifies a modification to BIP-380 key expressions allowing a single descriptor to cover multiple derivation path steps simultaneously (e.g., receive and change paths).

## Body

### Zweck

In standard HD wallet usage (BIP-44/84/86), an account has two derivation branches: external (receive) at `.../0/*` and internal (change) at `.../1/*`. Currently, two separate descriptors are needed to represent this: one for each branch.

BIP-389 introduces a `<m;n>` notation in key expressions that specifies multiple alternative derivation steps. `xpub.../<0;1>/*` means "derive using path /0/* for receive addresses and /1/* for change addresses." A single descriptor covers the entire account.

This makes descriptor backups more compact and reduces the chance of missing the change path when importing a wallet. It also aligns with BIP-388's account concept. The notation can include more than two alternatives for advanced use cases.
