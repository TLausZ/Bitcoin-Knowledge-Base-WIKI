# BIP-0379: Miniscript

## Summary

Miniscript is a language for writing Bitcoin Script in a structured, analysable way. It enables safe composition of spending policies, automated witness generation, and formal reasoning about script properties.

### Zweck

Bitcoin Script is expressive but difficult to reason about — it's easy to write a script that looks valid but has edge cases that allow theft, or that fails to spend correctly in unusual circumstances. Miniscript is a structured subset of Script where every valid Miniscript expression has provably correct properties.

A Miniscript policy describes spending conditions in a human-readable language: `thresh(2, pk(Alice), pk(Bob), pk(Carol))` means "any 2 of 3 sign." The Miniscript compiler converts this to optimised Script, checking correctness properties like: Is this script malleable? Can it be spent under any circumstance? What is the worst-case witness size (affecting maximum fee)?

These properties enable wallets to do things manually-written Script can't safely guarantee: automatic fee calculation, safe script composition, and interoperability between signers that don't need to trust each other's implementation.

Miniscript is widely implemented: Bitcoin Core, Ledger, Liana wallet, Sparrow, and others support it. BIP-379 formalises the language spec; the associated descriptor BIPs (380–390) integrate Miniscript into the descriptor language via `wsh(miniscript_expr)` and `tr(key, miniscript_expr)`.
