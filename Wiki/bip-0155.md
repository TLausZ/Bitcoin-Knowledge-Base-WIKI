# BIP-0155: addrv2 message

## Summary

A new P2P message to gossip longer node addresses over the P2P network, required to support Tor v3 hidden services, I2P, and other networks with addresses longer than 128 bits.

### Zweck

The existing `addr` P2P message stores node addresses as 128-bit fields — sufficient for IPv4 (32 bits, zero-padded) and IPv6 (128 bits), but too small for Tor v3 onion addresses (256 bits) or I2P addresses (256 bits). When Tor deprecated its v2 onion services in 2021, Bitcoin nodes running as Tor v3 hidden services had no way to announce their address over the network.

BIP-0155 defines a new `addrv2` message with variable-length address fields. It supports IPv4, IPv6, Tor v2 (legacy), Tor v3, I2P, and Cjdns, with a type byte indicating the network. Nodes negotiate `addrv2` support via the `sendaddrv2` message during handshake; old nodes continue using the original `addr`.

The practical effect: Bitcoin nodes can now operate as Tor v3 or I2P hidden services and have peers discover them, which is important for privacy and censorship resistance.
