# BIP-0074: Allow zero value OP_RETURN in Payment Protocol

## Summary

This BIP alters the Payment Protocol to allow for zero value OP_RETURN outputs in serialized PaymentRequests.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Autor(en):** Toby Padilla  
**Veröffentlicht:** 2016-01-29  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0074.mediawiki

### Zweck

BIP-0074 was a minor extension to BIP-70 that would have allowed merchants to include zero-value `OP_RETURN` outputs in their PaymentRequests. The motivation was to allow merchants to embed metadata (e.g. an invoice ID or memo) in the transaction without requiring the customer to pay an amount to that output.

BIP-70 at the time rejected outputs with zero value, making data-carrier outputs in merchant-specified transactions impossible. The proposal was closed, likely because BIP-70 itself was already in decline by the time this was written, making the extension unnecessary.
