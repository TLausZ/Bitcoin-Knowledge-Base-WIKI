# BIP-0134: Flexible Transactions

## Summary

This BIP describes the next step in making Bitcoin's transaction format more flexible and easier to extend, fixing all known cases of malleability and resolving significant technical debt.

## Body

**Typ:** Specification  
**Layer:** Consensus (hard fork)  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))

### Zweck

BIP-0134 proposed a complete redesign of the Bitcoin transaction format as a hard fork. It would replace the existing fixed serialisation with a token-based format (similar to TLV — type-length-value encoding), making the transaction structure extensible without future hard forks. It also aimed to fix malleability by separating witness data from the hashed transaction data.

The proposal was authored by Tom Zander, who developed it as part of the Bitcoin Classic/Bitcoin Unlimited movement. It represented the "big block" camp's preferred approach to SegWit — achieving the same goals (malleability fix, capacity increase) but via a clean hard fork rather than SegWit's soft fork compromise. It was closed when SegWit activated via soft fork in 2017.
