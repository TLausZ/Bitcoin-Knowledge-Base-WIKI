# BIP-0118: SIGHASH_ANYPREVOUT for Taproot Scripts

## Summary

This BIP describes a new type of public key for tapscript transactions. It allows signatures for these public keys to not commit to the exact UTXO being spent, enabling dynamic binding of transactions to different UTXOs with compatible scripts.

### Zweck

BIP-0118 introduces `SIGHASH_ANYPREVOUT` (APO), a new sighash flag for Tapscript signatures. Normally, a Bitcoin signature commits to the specific UTXO being spent (by txid and output index). APO relaxes this: the signature covers the transaction's inputs and outputs but not *which* UTXO is being spent. This means one signature can be reused to spend any UTXO with a compatible script.

The target application is the eltoo protocol (renamed LN-Symmetry) for Lightning Network channels. In current Lightning, when a counterparty broadcasts an old channel state, the honest party must monitor the chain constantly and respond quickly with a penalty transaction. LN-Symmetry replaces this with a simpler state machine where later states can always override earlier ones — no penalty needed, just update the state. APO signatures are required because the update transactions need to bind to "whichever state is currently on-chain" rather than a specific UTXO.

APO has also been proposed as the basis for transaction sponsors and certain vault designs. It remains in Draft with active development interest and is among the leading soft fork candidates after Taproot.
