# BIP-0375: Sending Silent Payments with PSBTs

## Summary

This document proposes additional PSBTv2 fields and updated role responsibilities for sending to Silent Payment (BIP-352) addresses.

### Zweck

Sending to a Silent Payment (BIP-352) address requires knowing the transaction's final inputs before computing the output key — because the Silent Payment output key depends on ECDH over the sender's input keys. In a PSBT workflow where inputs are assembled gradually, this creates a coordination problem: the output key can't be finalised until all inputs are known.

BIP-375 defines how to handle this in PSBTv2. It adds new PSBT fields for the recipient's Silent Payment code and the computed ECDH share. The Signer role is extended to also compute the ECDH shared secret contribution and add it to the PSBT. Once all signers have contributed their shares, the Finaliser can compute the final Silent Payment output key from all contributions, then sign.

A DLEQ proof (BIP-374) per input demonstrates to the recipient that the output key was computed correctly, without requiring the recipient to trust the sender.
