# BIP-0105: Consensus based block size retargeting algorithm

## Summary

A method of altering the maximum allowed block size of the Bitcoin protocol using a consensus based approach.

## Body

**Typ:** Specification  
**Layer:** Consensus (hard fork)  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Autor(en):** BtcDrak  
**Veröffentlicht:** 2015-08-21  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0105.mediawiki

### Zweck

BIP-0105 proposed a miner-signalling mechanism where the block size limit would increase only if a supermajority of miners (75%) voted for it in coinbase transactions over a 2016-block period. The increase could be at most 10% per period. Decreases could happen with a simple majority.

The "consensus based" framing was intended to give miners a formal governance role in block size changes, while limiting how fast the limit could grow. Critics noted the same concern as BIP-100: embedding miner voting in consensus rules gives miners ongoing policy power that users and node operators cannot easily override. Closed without activation.
