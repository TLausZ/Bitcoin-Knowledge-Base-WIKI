# BIP-0137: Signatures of Messages using Private Keys

## Summary

This document describes a signature format for signing messages with Bitcoin private keys, standardizing the format for cross-client compatibility.

### Zweck

BIP-0137 formalised the message signing format that Bitcoin wallets had been using informally since Satoshi's original client: sign an arbitrary message with a private key, produce a compact 65-byte signature that proves you control that key without revealing it.

The format: `\x18Bitcoin Signed Message:\n` + length-prefixed message is double-SHA256 hashed, then signed with ECDSA using the private key. The resulting signature is Base64-encoded. Because the signature includes a recovery parameter, verifiers can recover the signer's public key from the signature and message alone, then check it matches the claimed address.

BIP-137 extended the original format to support P2SH-P2WPKH, P2WPKH, and P2SH addresses alongside legacy P2PKH addresses. The limitation: it uses different prefix bytes for different address types, which is fragile. BIP-322 proposes a cleaner general scheme, but BIP-137 remains widely supported for legacy compatibility.
