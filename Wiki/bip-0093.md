# BIP-0093: codex32: Checksummed SSSS-aware BIP32 seeds

## Summary

This document proposes a new seed encoding format using a BCH error-correcting code, supporting Shamir's Secret Sharing and offline-only operations.

### Zweck

BIP-0093 (codex32) is a new seed encoding format designed for high-security self-custody, replacing the BIP-39 mnemonic approach with a base32 encoding using a BCH error-correcting code.

The error-detection capability is dramatically stronger than BIP-39. A codex32 share can detect up to 8 character errors or correct up to 4, entirely by hand using paper lookup tables. BIP-39 mnemonics have only a weak checksum detecting a single word error; they cannot correct errors at all. With codex32, a user can verify their backup is intact without a computer.

A seed can optionally be split into up to 31 shares with a configurable threshold (minimum 2) using Shamir's Secret Sharing. This is not bolted on — the entire scheme is designed so splitting, combining, and verifying shares are all operations that can be performed with pen, paper, and the published lookup tables, without ever entering the seed into a computer. An attacker who steals a single share learns nothing about the seed; recovering the seed requires the threshold number of shares.

Unlike BIP-39's optional passphrase, which creates ambiguity about whether the backup includes it, codex32 encodes exactly one secret. There is no "25th word" confusion.
