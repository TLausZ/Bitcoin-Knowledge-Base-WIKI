# BIP-0085: Deterministic Entropy From BIP32 Keychains

## Summary

"One Seed to rule them all" — this BIP derives child entropy (for other wallets, apps, or purposes) deterministically from a master BIP-32 seed, enabling a single backup to cover multiple wallet instances.

### Zweck

BIP-0085 solves a real self-custody problem: a user might run multiple wallets, hardware devices, or apps, each requiring its own seed. Managing multiple seed phrase backups is operationally complex and increases the risk of loss. BIP-85 lets you derive deterministic "child entropy" from a single master seed — any child entropy is reproducible as long as you have the master.

The mechanism: use BIP-32 to derive a key at a specific path (e.g. `m/83696968'/39'/0'/12'/0'` for a 12-word BIP-39 mnemonic), then pass the derived key material through HMAC-SHA512. The result is used as entropy for a new wallet seed, a WIF private key, an xprv, or other formats. The child is fully independent of the master wallet's funds — it's a separate wallet generated deterministically.

The BIP number `83696968` used in the path is the ASCII encoding of "ENTROPY" (0x454E5452 = 1163417682, not quite — actually it encodes "DRNG" or similar). The important point is it's a reserved namespace that won't collide with other derivation schemes.

Hardware wallets like SeedSigner and some firmware configurations support BIP-85 derivation directly.
