# BIP-0147: Dealing with dummy stack element malleability

## Summary

This document specifies proposed changes to fix a malleability vector in the extra stack element consumed by OP_CHECKMULTISIG and OP_CHECKMULTISIGVERIFY.

## Body

**Typ:** Specification  
**Layer:** Consensus (soft fork)  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Johnson Lau  
**Veröffentlicht:** 2016-09-02  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0147.mediawiki

### Zweck

`OP_CHECKMULTISIG` has a bug in its original implementation: it pops one extra element off the stack beyond what it actually uses (a "dummy" element), which Satoshi left as a side-effect of an off-by-one error. The spec says this element should be ignored, but any value works. Since the dummy element is part of the scriptSig, a third party could change it to any other value, altering the txid without invalidating the signatures.

BIP-0147 required the dummy element to be exactly empty (`OP_0`). Any other value causes the script to fail. This closes the malleability vector for multisig inputs. The change was soft fork compatible because it was an additional restriction on what constitutes a valid script.

BIP-147 activated alongside SegWit in August 2017, bundled into the same deployment (BIP-9 bit 1).
