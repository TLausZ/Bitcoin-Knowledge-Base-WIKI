# BIP-0022: getblocktemplate - Fundamentals

## Summary

This BIP describes a new JSON-RPC method for "smart" Bitcoin miners and proxies. Instead of sending a simple block header for hashing, the entire block structure is sent, and left to the miner to customize and assemble.

### Zweck

BIP-0022 replaced `getwork` as the mining RPC method. `getwork` sent miners only a block header to hash — the node handled all transaction selection and block assembly, leaving miners no agency over what went into their blocks. `getblocktemplate` sends the full block structure: the set of transactions to include, the coinbase transaction template, fee data, and the target. The miner can modify the coinbase (to add a payout address, for instance), optionally select a different transaction set, and then hash the assembled block.

This decentralisation of block assembly was significant: it meant miners were no longer forced to trust the node's transaction selection policy. It also enables things like merged mining (including data for auxiliary chains in the coinbase). `getblocktemplate` is still used by solo miners and some pool software, though Stratum V2 is its likely long-term successor.
