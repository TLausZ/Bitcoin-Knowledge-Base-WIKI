# BIP-0069: Lexicographical Indexing of Transaction Inputs and Outputs

## Summary

Currently there is no standard for bitcoin wallet clients when ordering transaction inputs and outputs. As a result, wallet clients often have a discernible blockchain fingerprint, and can leak private information about their users.

### Zweck

Before BIP-69, wallets each had idiosyncratic input/output ordering — some always put the change output last, some first, some ordered by value. These patterns acted as wallet fingerprints that blockchain analysts could use to identify which output was change and which wallet software created the transaction.

BIP-69 standardises a deterministic ordering: inputs are sorted lexicographically by (txid, vout) and outputs by (value, scriptPubKey). With deterministic ordering, change outputs are indistinguishable from payment outputs by position alone, improving privacy.

Adoption has been mixed. Some wallets (Samourai, Electrum) implemented it; others did not. The standard was also controversial because *uniform adoption* would itself become a fingerprint — a transaction visibly sorted per BIP-69 reveals it was built by a BIP-69 wallet. The privacy benefit depends on enough wallets adopting it to form a large anonymity set.
