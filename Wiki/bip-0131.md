# BIP-0131: Coalescing Transaction Specification (wildcard inputs)

## Summary

This specification defines a new type of transaction with "wildcard" inputs that spend all UTXOs at a specified address in a single input.

## Body

**Typ:** Specification  
**Layer:** Consensus (hard fork)  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Themen:** bips
**Autor(en):** Chris Priest  
**Veröffentlicht:** 2015-11-30  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0131.mediawiki

### Zweck

BIP-0131 proposed a new transaction type with "wildcard" inputs — instead of specifying individual UTXOs to spend, a single wildcard input would spend all UTXOs at a given address. The motivation was UTXO consolidation efficiency: if you have many small UTXOs at the same address, sweeping them currently requires listing each one as a separate input (and paying a fee proportional to the number of inputs). A wildcard input could reference an address and sweep everything in one compact input.

The proposal required a hard fork and also implicitly required address reuse (the wildcard targets an address, meaning multiple UTXOs must share it). The combination of hard fork + requiring address reuse made it a non-starter. Closed without adoption.
