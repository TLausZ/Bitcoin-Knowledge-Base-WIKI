# BIP-0062: Dealing with malleability

## Summary

This document specifies proposed changes to the Bitcoin transaction validity rules in order to make malleability of transactions impossible (at least when the sender doesn't choose to avoid it).

## Body

**Layer:** Consensus (soft fork)  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))

### Zweck

Transaction malleability is the ability to change a transaction's txid without invalidating its signatures. This was Bitcoin's most pressing bug in 2014 — it was famously (and inaccurately) cited by Mt. Gox as contributing to its collapse. The problem: DER-encoded ECDSA signatures had multiple valid encodings, so a third party could alter the signature bytes, change the txid, and relay the modified transaction. The original transaction's txid would then never confirm, breaking any dependent transaction that referenced it.

BIP-0062 proposed a comprehensive set of rules to eliminate malleability: strict DER encoding, low-S signatures, no unnecessary pushdata opcodes, and more. It was withdrawn because SegWit (BIP-141) fixed malleability more completely and more elegantly by moving signatures outside the hashed transaction data entirely — SegWit outputs' spending signatures are committed to separately, so altering them doesn't change the txid. Parts of BIP-62 that were still useful (strict DER signatures) were activated separately as BIP-66.
