# BIP-0174: Partially Signed Bitcoin Transaction Format

## Summary

This document proposes a binary transaction format (PSBT) which contains the information necessary for a signer to produce signatures for a transaction and holds signatures while the transaction is incomplete. Signers can be offline.

### Zweck

PSBT (Partially Signed Bitcoin Transaction) solves a coordination problem: how do you pass an unsigned or partially-signed transaction between multiple parties (hardware wallets, air-gapped signers, multisig cosigners) without a shared network connection?

The format is a binary container with two main sections. The **global** section holds the unsigned transaction. **Per-input** sections hold everything a signer needs to sign each input: the UTXO being spent, the redeem script (for P2SH), the witness script (for P2WSH), derivation paths for keys, and any partial signatures already collected. **Per-output** sections hold derivation paths for change outputs, allowing signers to verify change outputs belong to the wallet.

Four roles are defined: **Creator** (builds the unsigned transaction), **Updater** (adds UTXO data and scripts), **Signer** (adds signatures using available keys), and **Finalizer** (assembles complete scriptSigs/witnesses once all signatures are present). A **Combiner** can merge PSBTs from multiple signers.

This separation enables hardware wallet signing without an internet connection, multisig workflows where signers never interact directly, and watch-only wallet coordination. Virtually every serious Bitcoin wallet now supports PSBT.

BIP-370 (PSBTv2) extends the format with additional fields; BIP-371, BIP-372, BIP-373, BIP-375, BIP-376 add further fields for Taproot, CoinJoin, and other use cases.
