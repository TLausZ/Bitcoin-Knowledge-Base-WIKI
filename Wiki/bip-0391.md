# BIP-0391: Binary Output Descriptors

## Summary

This document proposes an efficient binary container format for encoding output descriptors, using key-value maps similar to BIP-174 PSBT.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Autor(en):** SeedHammer  
**Veröffentlicht:** 2026-01-27  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0391.mediawiki

### Zweck

Text-based descriptors are human-readable but inefficient for encoding in QR codes or on constrained devices (hardware wallets with small memory). BIP-391 proposed a binary encoding of descriptors using the same TLV (type-length-value) map format as PSBT, making it compact and easy to parse.

The format separated the descriptor template (structural policy) from key data, similar to BIP-388's wallet policies. It included metadata fields for label, creation timestamp, and network.

BIP-391 was closed — the community converged on PSBT-based descriptor exchange formats and BIP-388 wallet policies as the standard, making a separate binary descriptor format redundant.
