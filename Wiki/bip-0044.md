# BIP-0044: Multi-Account Hierarchy for Deterministic Wallets

## Summary

This BIP defines a logical hierarchy for deterministic wallets based on BIP-0032 and the purpose scheme described in BIP-0043.

## Body

**Typ:** Specification  
**Layer:** Applications  
**Status:** Deployed (aktiviert und im Einsatz)

### Zweck

BIP-0044 defines the standard HD path structure used by legacy (P2PKH) Bitcoin wallets: `m/44'/coin_type'/account'/change/index`.

Each level has a specific role. `44'` is the purpose (from BIP-43). `coin_type'` separates Bitcoin from other coins (`0'` for Bitcoin mainnet, `1'` for testnet). `account'` provides isolated accounts within a single seed — account `0'` and account `1'` share no keys, enabling separate wallets from one backup. `change` is `0` for external (receiving) addresses and `1` for internal change addresses. `index` is the sequential address counter.

Hardened derivation on the first three levels (`44'`, `coin_type'`, `account'`) means a leaked child private key cannot compromise the parent. The `change` and `index` levels are non-hardened, so the account's xpub can derive all receiving and change addresses — useful for watch-only wallets.

BIP-44 addresses begin with `1` (P2PKH). For SegWit wallets, BIP-49 (P2SH-wrapped, `m/49'`) and BIP-84 (native bech32, `m/84'`) define equivalent structures with the appropriate purpose field.
