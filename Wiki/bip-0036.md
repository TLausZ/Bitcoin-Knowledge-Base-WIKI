# BIP-0036: Custom Services

## Summary

This BIP adds new fields to the version message which clients can use to announce custom services without polluting the limited 64-bit service flags field.

## Body

**Typ:** Specification  
**Layer:** Peer Services

### Zweck

BIP-0036 attempted to solve the extensibility problem with Bitcoin's 64-bit service flags field in the `version` message. The service flags field (used to advertise node capabilities like `NODE_NETWORK`, `NODE_BLOOM`, etc.) has only 64 bits, so at most 64 distinct services could be announced. BIP-36 proposed adding a separate key-value map to the version message for custom service advertising.

The proposal was closed without adoption. The 64-bit flags field has proven sufficient in practice — only a handful of bits are in use. Custom service discovery has generally happened through other means (e.g. Tor hidden services, protocol-level negotiation, or out-of-band coordination).

## Open Questions

- Are there any current or planned services that would actually exhaust the 64-bit service flags field?
