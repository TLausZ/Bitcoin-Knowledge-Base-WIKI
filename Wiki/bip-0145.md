# BIP-0145: getblocktemplate Updates for Segregated Witness

## Summary

This BIP describes modifications to the getblocktemplate JSON-RPC call (BIP-22) to support segregated witness as defined by BIP-141.

## Body

**Typ:** Specification  
**Layer:** API/RPC  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Luke Dashjr  
**Veröffentlicht:** 2016-01-30  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0145.mediawiki

### Zweck

BIP-0145 updates the `getblocktemplate` RPC (BIP-22/23) to include SegWit-specific fields. It adds a `weightlimit` field replacing the old byte-based `sizelimit` (since SegWit blocks are measured in weight units, not bytes), a `txweight` field per transaction, and a `witness` field in each transaction object containing the witness data.

Mining software that implements `getblocktemplate` needed these updates to correctly assemble SegWit blocks — specifically to calculate block weight rather than block size, and to include witness data in coinbase commitments. Without BIP-145, mining pools using `getblocktemplate` could not produce valid SegWit blocks.
