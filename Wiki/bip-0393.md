# BIP-0393: Output Script Descriptor Annotations

## Summary

This document specifies an optional annotation syntax for output script descriptors, using URL-like query string delimiters to convey operational metadata such as scan start height or gap limit.

### Zweck

A descriptor alone doesn't tell a recovering wallet where to start scanning the blockchain or how many consecutive unused addresses to check (the gap limit). Without this metadata, recovery software uses conservative defaults — scanning from genesis and using a large gap limit — which is slow and wasteful.

BIP-393 adds a `?key=value&key=value` suffix to descriptors for optional metadata. Defined fields include `height` (block height to start scanning), `gap` (gap limit for address discovery), and `label` (a human-readable wallet name). Example: `wpkh(xpub.../0/*)#checksum?height=840000&gap=20`.

These annotations are purely advisory — they don't affect the scripts produced by the descriptor and don't change the checksum. Wallets that don't support annotations can safely ignore them.
