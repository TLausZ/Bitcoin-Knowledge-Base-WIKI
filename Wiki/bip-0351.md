# BIP-0351: Private Payments

## Summary

This BIP makes it possible for two parties to transact using addresses that only they can calculate, using on-chain methods only and minimizing blockchain footprint. Receiving parties can share their payment codes publicly without losing privacy.

### Zweck

BIP-351 is a reusable payment code scheme, similar in goal to BIP-47 (PayNym) but with a lighter on-chain footprint. The idea: you publish a static payment code, and senders can derive unique addresses for you from it without any communication. Each sender computes a unique address for each payment using an ECDH shared secret between their key and your payment code. No notification transaction is required (unlike BIP-47).

The approach uses a single P2TR output as a "notification" embedded in a normal transaction — specifically by using the OP_RETURN output or spending to an address derived from the shared secret. This avoids BIP-47's dedicated notification transaction, which is costly and leaves a privacy footprint.

BIP-351 lost attention after BIP-352 (Silent Payments) was published, which achieves similar goals with a cleaner cryptographic design and no notification output at all. Both remain Draft, but BIP-352 has more momentum.
