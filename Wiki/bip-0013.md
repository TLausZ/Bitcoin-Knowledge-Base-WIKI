# BIP-0013: Address Format for pay-to-script-hash

## Summary

This BIP describes a new type of Bitcoin address to support arbitrarily complex transactions. An address encoded under this proposal represents the encoded hash of a script, rather than the encoded hash of an ECDSA public key.

### Zweck

BIP-0013 defines the P2SH address encoding — the Base58Check addresses that start with `3`. While BIP-16 defines the consensus rules for P2SH outputs, BIP-13 defines what users actually see: the address format.

The encoding works by hashing the redeem script with HASH160 (SHA256 then RIPEMD160), prepending version byte `0x05`, and Base58Check-encoding the result. The `3` prefix comes from that version byte. A sender paying to a `3...` address doesn't need to know or care what the underlying script is — it could be multisig, a time-locked script, or anything else. The recipient reveals the script only when spending.

This separation between "what you pay to" (the hash) and "what the spending conditions are" (the script) was a significant UX simplification for complex transaction types.
