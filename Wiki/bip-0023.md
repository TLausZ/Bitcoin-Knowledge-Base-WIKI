# BIP-0023: getblocktemplate - Pooled Mining

## Summary

This BIP describes extensions to the getblocktemplate JSON-RPC call to enhance pooled mining.

## Body

**Typ:** Specification  
**Layer:** API/RPC  
**Status:** Deployed (aktiviert und im Einsatz)  
**Autor(en):** Luke Dashjr  
**Veröffentlicht:** 2012-02-28  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0023.mediawiki

### Zweck

BIP-0023 extends BIP-22's `getblocktemplate` with features needed for mining pools. It adds a work ID system so pools can track which template a submitted share came from; a "proposal" mode where a miner submits a full candidate block for the pool to validate before work begins; and a `mutable` field listing which parts of the template the miner is permitted to change (e.g. whether the transaction set, the coinbase, or the timestamp can be modified).

These extensions matter because pools coordinate work across many miners and need to ensure submitted work is valid. The proposal mode in particular prevents miners from doing wasted work on invalid block templates. BIP-23 brought getblocktemplate close to parity with Stratum (the dominant proprietary protocol at the time) for pool use cases.
