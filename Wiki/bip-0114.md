# BIP-0114: Merkelized Abstract Syntax Tree

## Summary

This BIP defines a new witness program type that uses a Merkle tree to encode mutually exclusive branches in a script, hiding unexecuted branches and enabling complex redemption conditions.

### Zweck

BIP-0114 was the original MAST (Merkelized Abstract Syntax Tree) proposal for Bitcoin. Instead of revealing all possible spending conditions in a script (which wastes block space and leaks information about unexecuted paths), you commit to a Merkle tree of script branches. When spending, you reveal only the branch you're using and a Merkle proof that it's part of the committed tree.

This enables scripts with many conditional branches (e.g. "spend with key A after 1 year, OR spend with keys A+B any time, OR spend with keys A+B+C immediately") without putting all branches on-chain. Only the executed branch is revealed.

BIP-114 was superseded by Taproot (BIP-341), which achieved the same MAST functionality in a more elegant way — Taproot integrates the Merkle tree directly into the public key tweaking mechanism, so a single-key spend reveals nothing about any script branches. BIP-114's approach was less private and more complex.
