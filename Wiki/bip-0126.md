# BIP-0126: Best Practices for Heterogeneous Input Script Transactions

## Summary

This BIP proposes best practice guidelines to minimize uncontrolled disclosure of personally identifiable information in transactions containing inputs from different address types or wallets.

### Zweck

When a transaction spends inputs from multiple address types (e.g. mixing a legacy `1...` address with a SegWit `bc1q...` address), it leaks information: blockchain analysts can infer that all these inputs belong to the same wallet, and the unusual combination can also fingerprint the wallet software that constructed the transaction.

BIP-0126 identifies several specific patterns that leak information — mixing script types, using a different output script type than the inputs — and recommends wallets either avoid these combinations or handle them in standardised ways to reduce fingerprinting. It also recommends wallets avoid mixing UTXOs from different wallet derivation accounts in a single transaction.

This is a companion to BIP-69 (lexicographic ordering): both address different aspects of transaction-level privacy. BIP-126 remains in Draft, reflecting that these are guidelines rather than enforced protocol rules.
