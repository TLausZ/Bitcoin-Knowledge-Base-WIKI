# BIP-0321: URI Scheme

## Summary

This BIP proposes a URI scheme for describing Bitcoin payment instructions, replacing and updating BIP-21.

### Zweck

BIP-0321 updates the Bitcoin URI scheme (originally BIP-21) for the modern address landscape. BIP-21 defined `bitcoin:ADDRESS?amount=X&label=Y&message=Z` — a URI format that wallets can open from QR codes or links to pre-populate payment details. But BIP-21 was written for legacy P2PKH addresses and doesn't cleanly accommodate multiple address types, Lightning invoices, or other payment methods.

BIP-321 extends the scheme to support multiple payment instructions at once. A single URI can include a fallback on-chain address, a Lightning BOLT11 invoice, a BOLT12 offer, a Silent Payments address (BIP-352), or other payment types — each as a separate URI parameter. The wallet picks the best option it supports. A Lightning-capable wallet pays via Lightning; an on-chain-only wallet uses the fallback address. This makes QR codes forward-compatible with new payment methods.
