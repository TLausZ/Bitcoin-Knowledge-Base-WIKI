# BIP-0310: Stratum protocol extensions

## Summary

This BIP provides a generic mechanism for specifying Stratum protocol extensions, with version rolling — allowing miners to roll bits in the nVersion field of block headers — as the primary use case.

### Zweck

Bitcoin mining requires hash exhaustion over the 32-bit nonce field. At modern hashrates, ASICs exhaust the entire 4-billion-value nonce space in milliseconds, forcing them to also roll the timestamp or rely on the pool to change the coinbase (the extranonce). These workarounds add latency or reduce efficiency.

BIP-310 proposes "version rolling" as a Stratum extension: mining pools tell miners which bits in the nVersion field they're allowed to freely vary. With BIP-320 reserving specific nVersion bits for this purpose, miners gain additional nonce space in the block header itself — up to 16 bits (BIP-320) or 24 bits (BIP-323). This allows header-only mining without frequent coinbase changes.

BIP-310 also defines a generic extension negotiation mechanism so future Stratum extensions can be added without breaking existing clients.
