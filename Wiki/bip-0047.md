# BIP-0047: Reusable Payment Codes for Hierarchical Deterministic Wallets

## Summary

This BIP defines a technique for creating a payment code which can be publicly advertised and associated with a real-life identity without creating the loss of security or privacy inherent to P2PKH address reuse.

### Zweck

BIP-0047 (known as "PayNym" in the Samourai wallet ecosystem) solves a fundamental tension in Bitcoin privacy: sharing a static identifier with someone so they can pay you repeatedly, without reusing a single address. Address reuse links all payments together on-chain.

The mechanism uses a one-time "notification transaction" to establish a shared secret. The sender derives a shared secret from their private key and the recipient's public payment code (via ECDH — Elliptic Curve Diffie-Hellman), then sends a small on-chain notification transaction that encodes the sender's payment code in an OP_RETURN output. After this setup, both parties can independently derive the same sequence of unique addresses for future payments, without any further on-chain coordination.

The notification transaction is BIP-47's main drawback: it costs on-chain fees and publicly associates the two parties' payment codes on-chain. Silent Payments (BIP-352) improves on this by eliminating the need for a notification transaction entirely, at the cost of requiring the recipient to scan more of the blockchain.
