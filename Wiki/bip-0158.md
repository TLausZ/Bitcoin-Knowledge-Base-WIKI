# BIP-0158: Compact Block Filters for Light Clients

## Summary

This BIP describes a structure for compact filters on block data, for use in the BIP-157 light client protocol. The filter construction uses Golomb-Rice coding for compression.

### Zweck

BIP-0158 defines the filter construction algorithm used by BIP-157. For each block, a compact filter is built by hashing all relevant items in the block (input outpoints, output scripts, and coinbase scriptPubKeys) into a Golomb-Coded Set — a probabilistic data structure similar to a Bloom filter but with better compression.

The basic "type 0" filter encodes every scriptPubKey being created or spent in the block. A light client hashes its own addresses/scripts using the same procedure, then checks whether they appear in the filter. False positive rate is approximately 1/M where M is configurable; BIP-158 uses M=784931 (roughly 1-in-800k false positive per filter). This means a client checking 1000 blocks will download ~1.3 extra blocks on average due to false positives.

The filter for each block is about 25KB on average for a typical block. For a full blockchain sync, a client downloads ~60GB of filters instead of ~600GB of full blocks — a 10x reduction, while maintaining much better privacy than BIP-37 Bloom filters.
