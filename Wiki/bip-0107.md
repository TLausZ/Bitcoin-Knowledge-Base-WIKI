# BIP-0107: Dynamic limit on the block size

## Summary

This BIP proposes a dynamic limit to the block size based on transaction volume.

## Body

**Typ:** Specification  
**Layer:** Consensus (hard fork)  
**Status:** Closed (geschlossen (zurückgezogen oder ersetzt))  
**Themen:** bips
**Autor(en):** Washington Y. Sanchez  
**Veröffentlicht:** 2015-09-11  
**Quelle:** https://github.com/bitcoin/bips/blob/master/bip-0107.mediawiki

### Zweck

BIP-0107 proposed a dynamic block size limit calculated from the median transaction volume of the previous 2016 blocks. The limit would be set to double the median block size, capped at 8MB. The aim was a market-driven limit: if most blocks were small, the cap would be low; if blocks were consistently large, the cap would grow.

One distinctive feature: the proposal included a floor of 1MB (no contraction below the starting point) and a ceiling, making it an asymmetric ratchet rather than a pure dynamic limit. Closed without activation alongside the rest of the block size proposal wave.
