# BIP-0178: Version Extended WIF

## Summary

An extension to the Wallet Import Format (WIF) to specify what kind of Bitcoin address the private key corresponds to.

### Zweck

WIF (Wallet Import Format) is the Base58Check encoding of a raw private key. The standard WIF format carries one bit of information beyond the key itself: whether the corresponding public key is compressed or uncompressed (via a `0x01` suffix byte). This determines whether the key corresponds to a legacy P2PKH address using compressed or uncompressed pubkeys.

But with SegWit, a single private key can correspond to multiple address types: legacy P2PKH, P2SH-P2WPKH, and native P2WPKH — all using the same underlying key. Importing a WIF key into a wallet leaves ambiguity about which address type was intended, causing wallets to potentially look for funds at the wrong address.

BIP-178 proposes extending the suffix byte to encode the address type explicitly, using different values for each output type. This is still Draft status — the problem it addresses is real, but the ecosystem has largely moved toward PSBT and descriptor-based wallet import (BIP-380+) as the preferred solution for unambiguous wallet import.
